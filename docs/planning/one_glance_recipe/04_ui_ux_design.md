# 📱 한눈 레시피 - UI/UX 디자인 설계

## 1. 페이지 구조 (A안: 단일 스크롤 구조)
탭 전환 없이 한 화면에서 모든 입력을 완료하는 **단일 스크롤** 구조를 지향합니다.

### 레이아웃
- **상단 고정**: 제목 입력 + "새 레시피" 버튼
- **본문 스크롤**:
  - 재료 영역 (카드형 블록, 중복 체크, 자동 스크롤 이동)
  - 조리 단계 영역 (단계별 카드, 조리법 선택, 재료 체크, 세부 옵션, 실시간 문장 미리보기)
- **하단 고정**: [이미지로 저장하기] [카카오톡 공유] 버튼

## 2. 핵심 UX 정책
- **Mobile First**: 터치 영역 44px 이상, 본문 16px 이상
- **입력 최소화**: 숫자 직접 입력보다 버튼형 UI 중심
- **기본값 적극 활용**: 사용자는 수정만 하면 되도록 자동 선택 제공
- **자동 포커싱**: 재료/단계 추가 시 해당 위치로 부드럽게 스크롤
- **실시간 결과 확인**: 각 단계 하단에 자동 생성된 문장을 즉시 표시

## 3. 반응형 대응
- **데스크톱**: 재료 영역과 단계 영역을 좌우로 분할 배치하여 공간 효율성 증대 가능
- **모바일**: 상하 배치 유지 및 하단 버튼 고정

## 4. 최종 레시피 카드 (결과물)
- **이미지 비율**: 4:5 (1080 x 1350)
- **구성**: 제목, 재료 목록 (재료+수량+단위), 조리 순서 (생성된 문장 나열)
- **스타일**: 가독성 중심, 불필요한 장식 배제
