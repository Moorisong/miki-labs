# 📦 하루상자 (HaruBox) 기획 개요

## 1. 서비스 컨셉
하루상자는 짧은 시간 안에 가볍게 즐길 수 있는 **아기자기한 놀이·게임·기록 콘텐츠를 모아둔 웹 플랫폼**입니다.

사용자는 하루상자 안에서:
- 게임을 즐기고
- 소소한 기록을 남기고
- 결과물을 저장하거나 공유하며
하루의 작은 재미를 수집합니다.

---

## 2. 핵심 방향성
- 로그인은 **선택적**
- 로그인 없이도 대부분의 콘텐츠 체험 가능
- 로그인 시 **기록·저장·개인화** 제공

---

## 3. 로그인 설계

### 3.1 로그인 원칙
- **초기 진입 허들은 낮게**
- "로그인 안 해도 됨" (Guest Play First)
- 단, 저장/히스토리/랭킹 등록은 로그인 필요

### 3.2 로그인 방식
- 소셜 로그인 위주 (Google, Kakao)
- 이메일/비밀번호 방식은 선택사항

### 3.3 로그인 시 제공 기능
- 플레이 기록 저장
- 개인 보관함 (이미지, 결과물)
- 콘텐츠별 통계
- 기기 변경 시 데이터 유지

---

## 4. 전체 페이지 구성

### 4.1 메인 페이지 (`/`)
**목적:** 하루상자의 분위기 전달 및 콘텐츠 탐색

*   **상단**:
    *   로고: 하루상자
    *   메뉴: 콘텐츠 | 로그인
*   **히어로 섹션**:
    *   문구: "오늘의 작은 재미를 담다"
*   **콘텐츠 카드 그리드**:
    *   한눈 레시피 (One-glance Recipe) - **NEW!**
    *   TOBY - 선생님을 위한 서비스 툴 (외부 링크)
    *   미니게임 (Coming Soon)
*   **하단**:
    *   서비스 설명
    *   SEO 텍스트

### 4.2 콘텐츠 목록 (`/contents` - 예정)
**목적:** 제공 중인 모든 놀이 콘텐츠 모음 (현재 메인 페이지에서 바로 접근 가능하도록 구현)

*   카드형 UI
*   상태 표시: NEW, 인기, 준비중
*   클릭 시 각 콘텐츠 상세/플레이 페이지로 이동

### 4.5 한눈 레시피 (`/recipe`)
*   **URL**: `/recipe`
*   **컨셉**: 재료와 조리 방법을 입력하면 자동으로 완성도 높은 문장형 레시피를 생성해주는 서비스
*   **주요 기능**:
    *   **최소 입력 시스템**: 재료 및 단위 선택, 조리 방법별 옵션 자동 선택
    *   **문장 생성 엔진**: 규칙 기반 자연스러운 문장 및 조사 처리
    *   **실시간 미리보기**: 입력 즉시 생성된 레시피 문장 확인 가능
    *   **저장 및 공유**: 4:5 비율 레시피 카드 이미지 저장 및 카카오톡 공유
    *   **데이터 정책**: LocalStorage 기반 3일(72시간) TTL 유지

### 4.6 TOBY (외부 링크)
*   **URL**: `https://toby-teacher.vercel.app`
*   **컨셉**: 학교 선생님들을 위한 서비스 툴 모음 사이트
*   **연동 방식**:
    *   메인 페이지 콘텐츠 카드에 노출 (외부 링크로 이동)
    *   네비게이션 바에 TOBY 항목 추가 (외부 링크로 이동)
    *   로고: 별도 제공 예정

### 4.6 마이페이지 (`/mypage`)
*   로그인 정보
*   콘텐츠 이용 통계
*   데이터 초기화
*   탈퇴

- 나만의 레시피 만들기

- 웹 레시피 서비스

## 6. 확장 고려 사항
- 콘텐츠 추가 시 `/contents/{slug}` 또는 `/{slug}` 구조 유연하게 유지 (현재 `/recipe` 사용)
- 모든 콘텐츠는 "하루상자" 세계관과 디자인 톤앤매너 공유
- 추후 모바일 앱(WebView 기반 등) 확장 가능 구조

## 7. 한 줄 요약
하루상자는 **하루의 소소한 재미를 담아두는 작은 놀이 상자**입니다.
