# 공유 기능 UX 명세 (카카오 공유 + 링크 복사)

## 🎯 목표
사용자가 **자연스럽게 결과를 확산**하도록 유도하면서  
요청 페이지에서도 **재방문 유입**을 확보한다.

핵심 원칙:
- 요청 페이지 → 참여 유도
- 결과 페이지 → 결과 확산

---

## 1. 페이지 역할 정의

| 페이지 | 공유 목적 | 공유 메시지 성격 |
|---|---|---|
| 요청 페이지 | 친구 초대 / 참여 유도 | "너도 해봐" |
| 결과 페이지 | 결과 자랑 / 확산 | "내 결과 봐봐" |

기능은 같지만 **목적이 다르다.**

---

## 2. 버튼 구조 설계 (중요)

버튼은 4개지만 사용자에게는 **2개처럼 보이게 설계**한다.

### 공유 영역 UI 구조
```
[ 공유하기 ]
카카오톡 공유
링크 복사
```

페이지마다 "공유하기" 섹션 1개만 존재한다.

사용자가 인식하는 버튼 수:
- 요청 페이지 → 공유하기 1개
- 결과 페이지 → 공유하기 1개

---

## 3. 요청 페이지 공유 명세 (참여 유도)

### 노출 위치
- 투표 버튼 아래
- 페이지 하단 CTA 영역

### 섹션 타이틀
친구랑 같이 해보기 👇

### 버튼 텍스트
- 카카오톡 공유 → 친구 초대
- 링크 복사 → 초대 링크 복사

### 카카오 공유 메시지 템플릿
```yaml
타이틀: 당신이 보는 저의 모습이 궁금합니다
설명: 10명의 평가로 완성되는 성격 테스트입니다. 잠시 시간을 내어 참여 부탁드립니다.
버튼: 참여하기
링크: 요청 페이지 URL
```

### 목적
- 신규 사용자 유입
- 친구 초대
- 트래픽 증가

= 마케팅 기능

---

## 4. 결과 페이지 공유 명세 (바이럴 핵심 - 탭 구조 도입)

### 🎯 공유 목적
결과 공유(자랑)와 친구 초대(확산) 두 가지 목적을 동시에 달성한다.
단, 버튼을 나열하지 않고 **탭 구조**를 통해 사용자에게는 항상 **2개의 버튼만 노출**한다.

### 노출 위치
- 결과 카드 바로 아래 (최상단 강조 영역)

### 섹션 타이틀
친구에게 공유하기 🔥

### 탭 구조
`[ 내 결과 공유 ]` (기본 선택) `[ 친구 참여시키기 ]`

---

### 모드 1 : 내 결과 공유 (기본)
- **목적**: 결과 자랑 / 바이럴
- **버튼**: [카카오톡 공유] / [링크 복사]
- **카카오 메시지**:
  ```yaml
  타이틀: 숨겨진 성격 결과가 공개되었습니다
  설명: 주변 사람들이 본 나의 모습까지 포함된 결과입니다. 생각보다 정확하고, 조금 놀랄 수도 있습니다.
  버튼: 결과 확인하기
  ```
- **링크**: 결과 페이지 URL (개인 결과 포함)

---

### 모드 2 : 친구 참여시키기
- **목적**: 친구 유입 / 참여 증가
- **버튼**: [카카오톡 공유] / [링크 복사]
- **카카오 메시지**:
  ```yaml
  타이틀: 당신이 보는 저의 모습이 궁금합니다
  설명: 10명의 평가로 완성되는 성격 테스트입니다. 잠시 시간을 내어 참여 부탁드립니다.
  버튼: 참여하기
  ```
- **링크**: 요청 페이지 URL (친구 응답 페이지: `/htsm/answer/[id]`)

---

## 5. 성장 루프 구조

`결과 공유 & 친구 초대 → 친구 클릭 → 참여 → 결과 생성 → 다시 공유`

이 루프가 서비스 확산의 핵심이다.

---

## 6. UX 혼란 방지 규칙 (중요)

### 반드시 지켜야 할 것
- **결과 페이지에서는 탭 구조 사용** (버튼 4개 나열 금지)
- 공유 버튼은 항상 2개만 노출
  - 카카오톡 공유
  - 링크 복사

### 절대 하면 안 되는 것
- 탭 없이 버튼 4개를 한 번에 노출 ❌
- 모드 구분 없는 일반 공유 버튼 배치 ❌

---

## 7. 최종 정리

| 페이지 | 사용자 인식 버튼 수 | 실제 기능 |
|---|---|---|
| 요청 페이지 | 공유하기 1개 | 친구 초대 (단일 모드) |
| 결과 페이지 | 공유하기 1개 | **내 결과 공유 + 친구 초대 (탭 선택)** |

👉 기능은 2배로 확장
👉 UX 복잡도는 그대로 유지
