# 🧩 통합 기능 구현 현황 (Integrated Features)

현재까지 구현된 주요 기능과 시스템 연동 현황입니다.

## 1. 닉네임 시스템 (Nickname System)
사용자의 닉네임 관리 및 제한 정책이 적용되었습니다.

### 1.1 닉네임 수정
*   **접근**: 네비게이션 바(NavBar)의 닉네임/프로필 영역 클릭 시 수정 모달 진입.
*   **정책**:
    *   **글자 수 제한**: 2~10자. (한글 입력 시 조합 문자 초과 방지를 위한 강제 `slice` 적용)
    *   **변경 주기 제한**: 닉네임 변경 후 **30일(1개월)** 동안 재변경 불가.
    *   **중복 확인**: API를 통해 실시간 중복 확인.

### 1.2 UI/UX
*   **모달(Modal)**:
    *   **수정 모드**: 기존 닉네임이 입력된 상태로 열림.
    *   **제한 안내**: 변경 불가 기간인 경우 남은 일수를 안내하고 수정 불가능하도록 잠금 처리.
    *   **드래그 오동작 방지**: 모달 내부에서 텍스트 드래그 중 실수로 모달 밖에서 마우스 클릭을 해제(MouseUp)하더라도 모달이 닫히지 않도록 `onMouseDown`/`onMouseUp` 검증 로직 추가.
    *   **모바일 오버레이**: 모바일 환경(768px 미만)에서는 배경 터치로 모달이 닫히지 않도록 하여 오조작 방지.

## 2. 랭킹 시스템 (Ranking System)
메인 화면과 랭킹 페이지에 실제 데이터베이스 데이터가 연동되었습니다.

### 2.1 메인 홈 (Home)
*   **Top 5 랭킹**: `scores` 컬렉션에서 점수(내림차순) 및 달성일(오름차순) 기준으로 상위 5명을 조회하여 표시.
*   **데이터 연동**: 하드코딩된 더미 데이터를 제거하고, 서버 컴포넌트(`Home`)에서 DB 직접 조회(`getDatabase`)를 통해 실시간 데이터 반영.
*   **예외 처리**: 데이터가 없을 경우 "랭킹 정보가 없습니다" 안내 문구 표시.

### 2.2 데이터 구조
*   **Collection**: `scores`
*   **Fields**: `nickname`, `score`, `createdAt` 등 사용.

## 3. UI/UX 개선 (Polish)
*   **네비게이션 바**: 닉네임에 마우스 오버 시 수정 가능함을 나타내는 아이콘(✏️)과 스타일 적용.
*   **반응형**: 모바일에서도 닉네임 수정 버튼이 적절한 크기와 스타일로 표시되도록 조정.

## 4. 게임 컨트롤 및 HUD (Game Controls & HUD)
게임 편의성과 조작감을 개선했습니다.

### 4.1 키보드 컨트롤 (WASD)
*   **입력 방식 변경**: 기존 `event.key` 방식에서 `event.code` 방식으로 변경하여, **한글 입력 상태**에서도 WASD 이동 키가 정상적으로 인식되도록 개선했습니다.
    *   `KeyW` / `ArrowUp`: 전진
    *   `KeyS` / `ArrowDown`: 후진
    *   `KeyA` / `ArrowLeft`: 좌이동
    *   `KeyD` / `ArrowRight`: 우이동
    *   `Space` / `Enter`: 시작 / 잡기

### 4.2 HUD 레이아웃 최적화
*   **상단 배치**: 시도 횟수와 점수를 화면 좌측 상단으로 이동하여 시선 이동 최소화.
*   **비로그인 안내**: 게임 대기(Idle) 상태일 때만 로그인 안내 문구를 노출하여 플레이 중 방해 요소 제거.
