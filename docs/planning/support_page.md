# 후원 페이지 기획안

## 1. 목적
이 사이트는 기본적으로 **완전 무료 서비스**이며, 후원은 서비스 발전을 위한 요청이 아니라 **자발적 응원**의 개념으로 제공합니다.
사용자가 **“이 서비스는 정말 무료로 운영되는구나”**라는 신뢰를 느끼도록 과한 요청 없이 담백한 UI/UX를 구성하는 것이 핵심 목표입니다.
- 후원을 강요하지 않는다.
- 무료 서비스라는 인상을 강화한다.
- 후원은 "선택적 응원"이라는 느낌을 준다.

## 2. 구조
- **후원 안내 문구 (상단)**: 가운데 정렬, 가독성을 고려한 넉넉한 줄 간격, 과도한 강조 제거.
  > 이 사이트는 무료로 운영되고 있습니다.
  > 아이들이 더 즐겁게 활동할 수 있는 작은 도구들을 취미처럼 만들고 있습니다.
  > 혹시 이 서비스가 마음에 들었고 응원해 주고 싶은 분이 계시다면 아래 후원하기 버튼을 클릭해 주세요.
  > 물론 후원 없이도 모든 기능은 언제든 자유롭게 사용하실 수 있습니다.

- **후원 버튼 (중앙)**: "커피 한 잔으로 응원하기 ☕"
- **후원 계좌 영역**: OO은행 123-4567-8910 홍길동
- **계좌 하단 안내 문구**: 
  > 후원은 익명으로도 가능합니다. 보내주신 마음은 감사히 받겠습니다 :)

## 3. UX (사용자 경험)
- **미니멀 & 담백함**: 후원 요청 느낌을 최소화. 팝업 후원 요청, 강한 상태의 CTA 디자인 및 과한 색상을 피하고 부드러운 색상을 사용합니다. 버튼 호버 시 살짝 밝아지는 정도로 처리.
- **인터랙션**: 
  - 버튼 클릭 시 후원 계좌 영역이 expand 되면서 계좌 정보 노출 (초기 상태 숨겨짐).
  - 계좌번호 옆 '[계좌번호 복사]' 버튼 제공. 
  - 복사 버튼 클릭 시 토스트 메시지로 "계좌번호가 복사되었습니다" 표시.
- **모바일 대응**: 텍스트 가독성 및 버튼 중앙 정렬 유지. 계좌 영역 expand 애니메이션 자연스럽게 적용.

## 4. 데이터
- 서버와의 통신 불필요. 
- 복사할 계좌 정보는 프론트엔드 레벨에서 상수로 관리하여 렌더링.

## 5. 구현
- React/Next.js를 기반으로 구현.
- Clipboard API (`navigator.clipboard.writeText`) 사용.
- Expand 애니메이션은 CSS transition으로 처리 (max-height 또는 grid-template-rows 팁 사용).
