# 💾 한눈 레시피 - 저장 및 공유 전략

## 1. 데이터 저장 정책
- **LocalStorage + 3일 TTL**
- 서버 및 DB 없이 브라우저 로컬 저장소 활용
- **TTL (Time To Live)**: 72시간 (3일) 경과 시 데이터 자동 초기화

## 2. 저장 방식 및 만료
- JSON 형태 저장: `title`, `ingredients[]`, `steps[]`, `generatedSentences[]`, `expireAt` 필드 포함
- 접근 시 `expireAt` 체크하여 만료 시 초기화 진행

## 3. 덮어쓰기 정책
- 최근 레시피 **1개만 유지**
- 새 레시피 생성 시 확인 모달 표시: "새 레시피를 생성하면 기존 내용이 삭제됩니다. 진행하시겠습니까?"

## 4. 저장 및 공유 로직
- **저장/공유 버튼 클릭 시**: 새창 광고 실행 로직 연동
- **광고 종료 후**:
  - 이미지 저장(기기 저장)
  - 카카오톡 공유 (로그인 필수)
- 기존 서비스들의 광고 플로우 로직 재사용

## 5. 저장 차단 조건 (필수 입력 검증)
- 재료 0개 또는 조리 단계 0개
- 단계 내 재료 선택 누락
- 문장 생성 실패 또는 빈 문장 발생 시
- 만료된 데이터 접근 시
- 토스트 알림으로 사용자 가이드 제공
