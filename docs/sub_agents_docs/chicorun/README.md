# 치코런 서브에이전트 병렬 작업 안내서

이 문서는 치코런 프로젝트를 여러 에이전트가 동시에 병렬로 분담하여 작업할 수 있도록 안내하는 메인 문서입니다.
**에이전트의 토큰 소모를 최소화**하기 위해 전체 프로젝트를 **완전히 독립된 4개의 도메인**으로 나누었습니다.

작업을 시작할 에이전트는 본인의 도메인에 해당하는 **아래의 개별 문서 중 하나만** 읽고 작업을 진행하십시오. 모든 문서를 읽을 필요가 없으므로 토큰 소모가 최소화됩니다.

---

## 🚀 에이전트별 작업 분리 리스트 (독립/병렬 작업 가능)

각 에이전트는 본인의 영역에만 집중해 코드를 완성해야 합니다.

### 1. 백엔드 전담 에이전트 (Backend Agent)
- **담당**: DB 스키마 설계(Student, Problem), JWT 인증, 문제 조회 및 정답 검정 로직, 글로벌 랭킹 API
- **문서 참조**: `docs/sub_agents_docs/chicorun/01_backend_agent.md`

### 2. 학생 학습/랭킹 프론트엔드 에이전트 (Frontend - Student & Ranking)
- **담당**: 학생 랜딩/로그인, 1~100레벨 학습 로직 엔진, 정답/오답/콤보 연출, 레벨 선택 모달, 글로벌 Top 30 랭킹 페이지
- **문서 참조**: `docs/sub_agents_docs/chicorun/02_frontend_student_agent.md`

---
> **병렬 개발 원칙 (필수 숙지)**
> - 각 도메인 간의 결합부(예: 꾸미기 컴포넌트를 랭킹에 표시할 때)는 우선 `Dummy Hook`이나 `Empty Component`를 껍데기만 만들어 사용합니다. 다른 에이전트의 완성을 기다리지 않습니다.
> - 상태 관리는 최대한 전역/서버 상태 도구를 이용해 독립적으로 선언합니다.
