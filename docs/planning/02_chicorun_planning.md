# 치코런(Chicorun)

## 서비스 기획

## 1. 서비스 개요

치코런 영어 학습 서비스는
교사가 클래스를 생성하고 학생들이 참여하여 **게임 방식으로 영어 학습을 진행하는 웹 서비스**입니다.
학생들은 닉네임과 비밀번호만으로 빠르게 접속하여 학습을 시작할 수 있으며
👉 **최초 로그인 이후에는 JWT 기반 인증으로 유지됩니다**
학습 진행 상황은 데이터베이스에 저장됩니다.
교사는 학생들의 학습 진행 상황을 확인할 수 있습니다.

---

## 2. 서비스 목표

- 교사가 수업에서 쉽게 활용 가능한 영어 학습 서비스 제공
- 학생들이 게임처럼 레벨을 클리어하며 영어 학습
- 로그인 절차 최소화
- 서버 부하 최소화
- 데이터 구조 단순화
- 초등 ~ 고등 단계별 어휘 학습 제공

---

## 3. 사용자 구조

### 👨🏫 교사

- 카카오 로그인 사용(하루상자에 이미 구현되어있음, 구현할 필요 없고 연동만)
- 클래스 생성 가능
- 학생 관리 가능
- 학습 진행 상황 확인 가능

### 👨🎓 학생

- 클래스 링크로 접속 (직접 코드 입력 대신 링크를 통한 입장)
- 닉네임 + 비밀번호 등록
- **학생은 하나의 클래스에만 속할 수 있음**
- 닉네임 2~10글자(한글/영문/숫자), 비밀번호 4~8글자(숫자만)
- 학습 진행

👉 **인증 방식**

- 최초 로그인 시 (닉네임 + 비밀번호 검증)
- 이후 **JWT 발급 및 사용 (Stateless 인증)**

```
JWT payload
req.user = {
  studentId,
  classCode
}
```

## 7. 🎯 레벨 선택 시스템

### 요구사항
* 학생은 언제든지 레벨 선택 가능 (학습 전 / 학습 중 모두)
* 레벨은 3단계로 구성: 초급, 중급, 고급

### 레벨 정의 및 범위
| 레벨 | 라벨 | 설명 | 문제 범위 (progressIndex) |
|---|---|---|---|
| beginner | 초급 | 기초 단어와 간단한 문장 (3~5단어, 현재형 중심) | 0 ~ 2999 |
| intermediate | 중급 | 시제 활용 및 문장 확장 (5~8단어, 과거 포함) | 3000 ~ 6999 |
| advanced | 고급 | 수능형 문장 및 복합 구조 (7~12단어, 응용/해석 중심) | 7000 ~ 9999 |

### 레벨 선택 동작 방식
* 학생이 레벨 선택 시: 해당 레벨의 **문제 범위로 progressIndex 재매핑**
* 기존 progressIndex는 유지하되, **문제 생성 시 level offset 적용**
* 총 문제 수: 10,000

---

## 8. 🧠 문제 생성 시스템

### 핵심 원칙
* 문제는 DB에 저장하지 않음 (서버 코드에서 생성)
* 항상 동일 입력 → 동일 문제 (Deterministic)
* `seed = hash(studentId + classCode + progressIndex)`
* `generateQuestion(progressIndex, seed)`

### 🧩 문제 유형 분배
| 유형 | 설명 |
|---|---|
| sentence_choice | 문장 선택형 |
| word_order | 단어 배열 |
| fill_blank | 빈칸 채우기 |
| translation | 번역 |
| transformation | 문장 변환 |
| error_detection | 틀린 문장 찾기 |
| word_meaning | 단어 의미 |

### 레벨별 유형 비율 (%)
| 유형 | 초급 (0-2999) | 중급 (3000-6999) | 고급 (7000-9999) |
|---|:---:|:---:|:---:|
| sentence_choice | 40 | 15 | 5 |
| word_order | 35 | 25 | 15 |
| fill_blank | 20 | 25 | 20 |
| translation | - | 20 | 35 |
| transformation | - | 10 | 15 |
| error_detection | 5 | 5 | 10 |
| word_meaning | - | - | - |

* **타입 결정 로직**: `type = getTypeByIndex(progressIndex)`
* 동일 유형 3회 연속 금지

### 🧱 템플릿 및 단어 풀 시스템
* 레벨별 단어 제한: beginner(기본), intermediate(과거형 포함), advanced(확장/구문)
* 오답 생성 규칙: 최소 2가지 오류 혼합 (시제, 수일치, 어순, 문법 누락, 유사 단어 등)

## 5. 문제 출제 방식 (핵심 수정)

### 5.1 기본 원칙

- **유형 비율은 고정**
- **문제는 랜덤 조회하지 않음 (중요)**

---

### 5.2 문제 생성 방식 (서버 부하 최적화)

👉 **문제는 DB에서 랜덤 조회 ❌**
👉 **progressIndex 기반으로 결정 ⭕**

```
questionId = generateQuestionId(level, progressIndex)
```

---

### 방식 설명

- 서버가 문제를 “조회”하는 게 아니라
    → **“계산해서 생성”**

---

### 특징

- DB 랜덤 쿼리 없음
- 캐싱 불필요
- 항상 동일 문제 보장
- 부하 거의 0

---

### 5.3 문제 구성 방식

👉 내부적으로는 아래 구조 유지
- 유형 비율 유지
- 난이도 흐름 유지
- 동일 유형 반복 제한

---

### 구현 방식

👉 progressIndex 구간으로 제어
예:
```
0~30 → 쉬운 문제
30~70 → 중간
70~100 → 어려움
```

---

👉 유형도 deterministic하게 결정

```
type = getTypeByIndex(progressIndex)
```

---

👉 실제 문제는 템플릿 기반 생성

```
generateQuestion(type, seed)
```

---

### 5.4 동일 유형 제한

- 동일 유형 3번 이상 연속 금지
👉 index 기반 로직으로 제어

## 12. 문제 템플릿 정의

### 📌 개요

- 템플릿 = 문제 생성용 고정 패턴
- 변수(단어, 문장)를 채워 다양한 문제 생성
- 동일 템플릿으로 수십~수백 문제 생성 가능

---

## 🎯 빈칸 채우기 템플릿 (12개)

### 1
```
{subject} ___ to {place}
```
### 2
```
{subject} ___ {food}
```
### 3
```
{subject} ___ every day
```
### 4
```
{subject} ___ yesterday
```
### 5
```
{subject} is ___ now
```
### 6
```
{subject} ___ at {time}
```
### 7
```
{subject} ___ with {person}
```
### 8
```
{subject} ___ in the {place}
```
### 9
```
{subject} ___ very {adjective}
```
### 10
```
{subject} can ___ {object}
```
### 11
```
{subject} wants to ___ {object}
```
### 12
```
{subject} ___ before {event}
```

---

## 📦 변수 정의

- `{subject}` : I, You, He, She, They
- `{verb}` : go, eat, play, study, make
- `{place}` : school, home, park
- `{food}` : apples, pizza, rice
- `{time}` : morning, night
- `{person}` : friends, teacher
- `{adjective}` : happy, fast, tired
- `{object}` : homework, soccer, dinner
- `{event}` : dinner, sleep

---

## ⚙️ 생성 구조

```
1. 템플릿 선택
2. 변수 랜덤 선택
3. 정답 생성
4. 오답 3개 생성
```

---

## 🎯 목적

- 문제 자동 생성 가능
- 난이도 제어 가능

## 13. 문제 길이 / 난이도 기준

### 📌 개요

- 문제 길이와 난이도는 **레벨 기반으로 통제**
- 문장 길이 = 난이도 체감에 직접적인 영향
- 일관된 기준으로 문제 품질 유지

---

### 🎯 레벨별 기준

### Level 1 ~ 30 (초등)

```
- 단어 수: 3 ~ 5
- 시제: 현재형 중심
- 구조: 단순 주어 + 동사 + 목적어
```
예:
```
I eat apples
She goes home
```

---

### Level 31 ~ 70 (중등)

```
- 단어 수: 5 ~ 8
- 시제: 현재 / 과거 혼합
- 구조: 부사, 시간 표현 포함
```
예:
```
I ate dinner yesterday
She plays soccer every day
```

---

### Level 71 ~ 100 (고등)

```
- 단어 수: 7 ~ 12
- 시제: 완료형 / 진행형 포함
- 구조: 접속사, to부정사 포함
```
예:
```
I have finished my homework
She is studying to pass the exam
```

---

### ⚙️ 적용 규칙

- 템플릿 선택 시 레벨 기준 필수 반영
- 단어 풀도 레벨에 맞게 제한
- 길이 초과 시 자동 제외 또는再生성

## 15. 오답 선택지 품질 규칙

### 📌 개요

- 오답은 “그럴듯하게 틀려야” 학습 효과와 몰입 상승
- 너무 쉬운 오답 → 게임성 붕괴
- 너무 어려운 오답 → 스트레스 증가

---

### 🎯 오답 생성 규칙

```
1. 시제 오류
- eat → ate / eating / eats

2. 수일치 오류
- He eat ❌
- She go ❌

3. 어순 오류
- I apples eat ❌

4. 유사 단어 혼동
- apple → apples
- go → goes

5. 문법 요소 누락
- She playing soccer ❌
```

---

### ⚙️ 생성 규칙

- 정답 1개 + 오답 3개 고정
- 오답 유형 최소 2개 이상 섞기
- 정답 위치 랜덤 배치

---

### 🎯 목적

- 학습 효과 강화
- 문제 신뢰도 상승
- 몰입 유지

- questionId만으로 검증하지 않음

서버:

- 동일 로직으로 문제 생성

- questionId + seed 전달

검증:
- questionId 일치
- seed 일치

---

## ✅ API 요청 수정

```
POST /api/answer

Headers:
Authorization: Bearer {token}

body:
- questionId
- seed
- answer
```

---

## ✅ 서버 로직 위치

👉 동일하게 solveController.js

---

### 코드 예시

```
const student = await Student.findById(req.user.studentId)

const currentIndex = student.progressIndex

const expected = generateQuestion(currentIndex)

// 🔐 핵심 검증
if (
  req.body.questionId !== expected.id ||
  req.body.seed !== expected.seed
) {
  return res.status(400).send('Invalid request')
}
```

## 🎯 개요

기존 뱃지 시스템을 확장하여
**유저가 직접 꾸미는 커스터마이징 시스템**을 도입한다.

- 뱃지 + 닉네임 + 영역 전체를 하나의 “프로필 카드”로 정의
- 게임적 요소 + 꾸미기 요소 결합
- 랭킹 페이지에서 시각적 경쟁 유도

---

## 1. 🏆 뱃지 구매 가격

- 모든 뱃지 가격: **300P로 고정**

---

## 2. 🎨 뱃지 커스터마이징 시스템

### 기능

유저는 뱃지에 다음 요소를 추가할 수 있다:

- 테두리 (Frame)
- 스티커 (Sticker)

---

### 🎯 동작 방식

- 유저가 스티커 선택 후 **클릭 위치에 배치**
- 드래그로 위치 이동 가능
- 저장 시 커스터마이징 상태 유지

---

### 💾 저장 구조 (중요)

이미지는 저장하지 않고, **구성 정보만 저장**

```json
{
  "badgeId": "gold_01",
  "frameId": "frame_gold",
  "stickers": [
    { "id": "star", "x": 10, "y": 20, "scale": 1, "rotate": 0 }
  ]
}
```
