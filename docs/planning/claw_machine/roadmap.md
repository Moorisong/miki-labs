# 🛣️ 개발 로드맵 (Roadmap)

## 0. 원칙
* **재미 우선**: 기술적 완성도보다 "뽑는 맛"이 중요
* **로그인 나중에**: 게임 로직부터 완성하고 붙인다.

## 1단계: 핵심 게임 플레이 (MVP)
* Three.js 환경 세팅
* Cannon-es 물리 엔진 연동
* 집게 이동 및 기본 잡기 로직 구현
* (UI 없이 게임만 돌아가게)

## 2단계: 튜닝 및 규칙
* 물리 파라미터 튜닝 (마찰력, 집게 힘 등)
* 연출 추가 (놓침, 성공 이펙트)
* 점수 계산 로직 확정

## 3단계: 시스템 연동
* [x] 백엔드 API 구축 (Next.js API Routes)
* [x] OAuth 로그인 연동 (NextAuth.js)
* [x] 랭킹 API 개발 및 연동

## 4단계: 폴리싱 (Polishing)
* [x] UI/UX 디자인 적용
    * [x] 닉네임 수정 시스템 (NavBar 연동, 1개월 제한)
    * [x] 모달 UX 개선 (드래그 방지, 모바일 오동작 방지)
    * [x] 시도 횟수/쿨타임 시스템 (localStorage 기반, 5회/1시간)
    * [x] 게임 HUD (시도 횟수, 쿨타임 표시, 비로그인 안내)
    * [x] 성공 시 로그인 유도 문구
* [x] 모바일 대응 (터치 컨트롤 및 레이아웃 최적화)
* [ ] 사운드 효과 (BGM, 효과음)

## 5단계: 배포 및 출시 (Launch)
* [ ] SEO 메타태그 최적화
* [ ] 최종 버그 테스트
* [ ] Vercel 배포

## 커밋 컨벤션 제안
* `feat`: 새로운 기능 (예: `feat: three.js claw base`)
* `fix`: 버그 수정
* `chore`: 설정 및 기타 (예: `chore: project setup`)
