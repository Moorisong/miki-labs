# 📘 튜토리얼 UX 기획

## 1. 노출 방식 (권장안)

**최초 1회 자동 노출 + 수동 재확인 버튼 제공**

- 게임 **첫 진입 시 자동으로 튜토리얼 모달 노출**
- 이후에는 홈 화면 또는 게임 화면에 **[게임 방법] / [?] 버튼**을 통해 언제든 다시 확인 가능

> 매번 강제로 보여주지 않음 (피로도 방지)

## 2. 노출 시점

- 조건: **튜토리얼 미확인 상태**
- 시점:
  - 홈 → 게임 진입 시
  - 또는 게임 시작 직전

## 3. 튜토리얼 확인 상태 관리

### 저장 위치
- **LocalStorage**
  - key 예시: `claw_tutorial_seen = true`

### 처리 로직
- 튜토리얼 모달을 끝까지 확인하거나 “다시 보지 않기” / “확인” 버튼 클릭 시 → `claw_tutorial_seen = true` 저장
- 이후 자동 노출 ❌

> 로컬스토리지 삭제 시 다시 노출되는 것은 **허용 범위** (보안/악용 대상 아님)

## 4. 튜토리얼 내용 구성 (간단 명확)

필수 포함 내용:
- 🕹️ **조작법**
  - **PC**: 방향키(이동) / 스페이스바(DROP)
  - **모바일**: 화면 버튼 터치
- 🎯 **게임 규칙**
  - “한 판당 기본 10회 시도”
  - “성공 시 남은 횟수 +1”
  - “시도 횟수 0 → 1시간 후 10회 충전” (Note: 기획서상 35회였으나 기본 시도 10회와 맞추거나 설정에 따름)
- 🕒 **쿨타임 규칙**
  - “횟수가 0이 될 때만 쿨타임 시작”
- 👤 **로그인 안내**
  - “로그인 없이도 플레이 가능”
  - **“로그인하면 기록을 랭킹에 저장할 수 있어요”**

## 6. 요약 원칙

- **강제 노출은 최초 1회만**
- **언제든 다시 볼 수 있는 접근성 제공**
- **로컬 기준 관리로 구현 단순화**
