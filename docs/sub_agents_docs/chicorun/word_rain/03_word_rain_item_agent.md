## 02. 워드레인 아이템 시스템 (시간 정지) 명세 (02_word_rain_item_system.md)
# 워드레인 아이템 시스템 (시간 정지) Planning

## 1. 목적
- 즉사 구조의 난이도를 완화하면서 플레이어의 실력(콤보)에 기반한 생존 전략 요소를 제공합니다.
- 실력 기반 보상을 제공하면서도 과도한 아이템 누적을 방지합니다.

## 2. 구조 (시스템 명세)
- **아이템명**: 시간 정지 (Time Freeze)
- **효과**: 5초 동안 모든 단어의 낙하 정지 (입력은 가능)
- **획득 조건**: `combo % 3 === 0` (콤보 3, 6, 9, ... 도달 시 1개 지급)
- **게임당 최대 지급 횟수**: 2회 (`maxItemEarnCount = 2`)
- **지급 횟수 기준**: 현재 보유 수량이 아닌, 지금까지 획득한 누적 횟수 기준으로 제한
- **게임 오버**: 단어 1개라도 바닥(빨간선)에 닿으면 즉시 종료

## 3. UX (사용자 경험)
- **UI 표시**: 아이템 보유 시 사용 버튼 노출, 현재 보유 수량 표시
- **사용 효과**: 버튼 클릭 시 즉시 발동, 5초간 시각적/기능적 정지 효과 제공
- **직관성**: 즉시 사용 가능한 간단하고 강력한 효과 유지

## 4. 데이터 (상태 관리)
```json
{
  "combo": "Number",
  "itemCount": "Number",
  "itemEarnedCount": "Number",
  "isTimeFrozen": "Boolean"
}
```

## 5. 지급 로직
```
if (combo % 3 === 0 && itemEarnedCount < maxItemEarnCount) {
  itemCount += 1
  itemEarnedCount += 1
}
```

## 6. 아이템 사용과 지급의 관계
- 아이템 사용 시: `itemCount -= 1`
- `itemEarnedCount`는 감소하지 않음
- 사용 여부와 관계없이 지급 횟수는 누적 유지
- 아이템 사용으로 추가 획득 기회를 만들 수 없음

## 7. 구현 가이드
- `onItemUse()`: `itemCount` 감소 및 `activateTimeFreeze(5s)` 호출
- `activateTimeFreeze(duration)`:
  - 모든 단어의 낙하 속도를 0으로 설정
  - `duration` 경과 후 원래 속도로 복구
- 콤보 처리 로직에 아이템 지급 로직 추가
- UI 컴포넌트에 아이템 사용 버튼 및 수량 표시부 추가

## 8. 설계 원칙
- 아이템은 실력(콤보)에 의해 획득
- 잘할수록 보상이 증가하지만 상한선 존재
- 아이템 사용으로 추가 획득 기회를 만들 수 없음
- 게임 밸런스를 유지하면서 성취감 제공

## AI 작업 지침
### 목적
워드레인 게임에 실력 기반 보상인 '시간 정지' 아이템 시스템을 구현하여 게임의 전략성과 생존성을 높인다.

### 작업 단계
1. **상태 관리 정의**: `combo`, `itemCount`(freezeCount), `itemEarnedCount`, `isTimeFrozen`(isFrozen) 상태 추가.
2. **아이템 획득 로직 구현**:
   - `combo % 3 === 0` 조건으로 아이템 지급.
   - `itemEarnedCount < maxItemEarnCount(2)` 조건으로 게임당 최대 지급 횟수 제한.
   - `itemEarnedCount`는 사용 시에도 감소하지 않는 누적 카운터.
3. **시간 정지 기능 구현**: `activateTimeFreeze(5s)` 함수 작성. 모든 단어의 속도를 0으로 만들고 일정 시간 후 복구.
4. **UI 업데이트**:
   - 아이템 보유 수량 표시 UI 추가.
   - 아이템 사용 버튼 추가 (보유 시 활성화).
   - 시간 정지 상태 시 시각적 피드백 제공.
5. **검증**: 콤보에 따른 아이템 지급, 사용 시 단어 정지 및 복구 로직 테스트.

### 주의사항
- 아이템 지급 조건은 `combo % 3 === 0` (3의 배수 콤보마다 지급).
- 게임당 최대 지급 횟수는 `maxItemEarnCount = 2`로 제한.
- 지급 제한은 현재 보유 수가 아닌 **누적 획득 횟수(`itemEarnedCount`)** 기준.
- 아이템 사용 시 `itemCount`만 감소, `itemEarnedCount`는 유지.
- 시간 정지 중에도 사용자의 입력(단어 선택)은 가능해야 함.
- 시간 정지 종료 후 단어들이 이전 속도로 정상적으로 낙하하는지 확인.
- **CSS 성능**: 매 프레임 위치가 업데이트되는 단어 요소에 `transition: all`을 사용하면 렉과 지연이 발생하므로 피할 것.
