# 03. 치코런 문제 템플릿 생성 및 관리 명세

## 1. 목표
- 레벨(1~100)마다 해당 레벨에 정확히 맞는 난이도의 문제를 생성.
- 문제가 순서대로 출제되며, 유형을 최대한 다양하게 구성.
- 모든 문제는 정답이 명확히 하나이며, 오류(구멍, 모호한 답, 오답 해설 오류 등)가 절대 발생하지 않도록 철저한 검증 규칙을 적용.

## 2. 문제 출제 방식
### 순차 출제 (Random X)
- 각 레벨별로 문제는 고정된 순서로 출제됨.
- 학생이 재접속하거나 레벨을 이동해도 `progressIndex + seed`에 따라 이전 문제 다음부터 정확히 이어서 출제됨.

### 레벨별 문제 수
- **초급 (1~30)**: 최소 12문제
- **중급 (31~70)**: 최소 15문제
- **고급 (71~100)**: 최소 18문제
- 한 번 클리어한 레벨은 순환하지 않고 다음 레벨로 이동 (복습 모드 별도 제공 예정).

## 3. 문제 유형 다양화
문제는 아래 유형을 레벨 구간별로 균형 있게 배분한다. 같은 유형이 연속으로 2번 나오지 않도록 순서를 설계.

### 초급 (1~30)
- `vocab` (단어 뜻)
- `basic_grammar` (시제, 전치사, 관계대명사)
- `simple_fact` (지문 속 사실 찾기)
- `synonym` (동의어)

### 중급 (31~70)
- `context_vocab` (문맥 속 어휘)
- `inference` (추론)
- `main_idea` (주제 파악)
- `purpose` (목적)
- `connector` (연결어/논리)
- `summary` (짧은 요약)

### 고급 (71~100) – 수능 유형 중심
- `long_inference` (장문 추론)
- `attitude` (필자 태도)
- `blank_grammar` (빈칸 문법+어휘)
- `order_arrange` (순서 배열)
- `comparison` (두 지문 비교)
- `true_false` (내용 일치/불일치)
- `integrated` (통합형: 지문 2개)
- `purpose_detailed` (세부 목적)

각 레벨당 유형 분포는 템플릿 생성 시 자동 균형 맞춰서 생성.

## 4. 문제 데이터 구조
```typescript
interface Problem {
  id: string;                    // UUID 또는 level-orderIndex
  level: number;                 // 1~100
  difficulty: 'easy' | 'medium' | 'hard';
  orderIndex: number;            // 같은 레벨 내 순서 (1부터 시작)
  passage: string;               // 지문 (고급은 200~450단어)
  question: string;              // 문제 stem
  choices: [string, string, string, string];  // 정확히 4개
  correctAnswerIndex: number;    // 0, 1, 2, 3 중 반드시 하나
  explanation: string;           // 오답 시 힌트 (2~3줄 MZ 말투, 정답 직접 언급 금지)
  questionType: string;          // 상단에 나열한 유형 중 하나
  wordCount: number;
}
```

## 5. 저장 방식
- 모든 문제를 미리 생성해서 서버 DB(또는 JSON 파일)에 저장.
- **이유**:
    - 용량 충분 (1,500문제 정도, 지문 포함 최대 5~8MB 수준).
    - 순차 출제를 위해 미리 만들어 놓는 것이 가장 안전하고 정확함.
    - 템플릿만 만들어 그때그때 생성하면 순서 보장 어려움, 난이도 편차, 오류 발생 위험 증가.
    - 나중에 문제 수정/보완 용이.

### 구현 방법
- 모든 문제는 `ChicorunProblemModel`을 통해 관리됩니다.
- 프론트엔드는 API `GET /api/chicorun/question?difficulty={difficulty}` 로 현재 진도(`progressIndex`)에 맞는 문제를 받아옵니다.
- 정답 제출은 `POST /api/chicorun/answer` 를 통해 처리됩니다.

## 6. 문제 생성 검증 규칙 (구멍 방지)
아래 조건을 자동 체크하며, 하나라도 위반되면 해당 문제를 재생성한다.
- 정답은 반드시 하나이며, 다른 선택지가 정답이 될 수 없음.
- 정답 선택지가 지문에 직접적·명확하게 근거가 있어야 함. (추측/모호 금지)
- 오답 선택지는 지문과 완전히 반대되거나, 지문에 없는 내용이거나, 부분적으로만 맞는 함정으로 구성.
- 빈칸 문제의 경우 정답이 문법·어휘상 유일하게 자연스러운 것.
- 수능형 문제는 실제 수능 해설 스타일로 검증.
- `explanation`은 오답일 때만 사용되며, 정답 시에는 절대 노출되지 않음.
- 지문과 문제 stem에 모순이 없음.
- 선택지 4개는 길이와 난이도가 비슷하게 구성.
- `explanation`은 MZ 말투 힌트 + 직관적 + 2~3줄. (정답 스포 금지!)

## 7. 오답 힌트 및 보상 규칙 (MZ 스타일)
- 최대 2~3줄 (한 줄당 40자 이하 권장).
- 말투: “어쩔”, “가보자고”, “폼 미쳤다”, “깔끼함”, “~함” 등 MZ 트렌드 반영.
- 정답 직접 언급 금지: "답은 2번이야" (X) -> "이 부분이 핵심인데 안 고르면 어쩔? 다시 봐보자고! 🔥" (O)
- 예시: “야 이건 지문에서 ‘however’ 뒤가 찐이야! 반대 의미인 거 체크함? 가보자고! �”
- 정답인 경우 해설 없이 바로 다음 문제로 이동 (기존 UX 유지).

### 시도 횟수별 및 난이도별 포인트 규칙
- **기본 보상 (1회차 기준)**: **5P**
- **시도 횟수 페널티**: 
  - 1차 시도 (Perfect!): **100% (5P)**
  - 2차 시도 (Great!): **60% (3P)**
  - 3차 시도 이상 (Good!): **20% (1P)**
- **난이도 및 레벨 차이 페널티**: 현재 최대로 검증된 레벨(`achievedMaxLevel`)보다 낮은 레벨을 풀거나, 추천 난이도보다 낮은 난이도를 선택한 경우 위 보상에 **factor(0.3~0.6)**가 곱해져 최소 1P까지 감소합니다.

## 8. 주요 API 리스트
1. `GET /api/chicorun/question`: 현재 진도 및 난이도에 맞는 문제 조회
2. `POST /api/chicorun/answer`: 정답 제출 및 포인트/진도/통계 업데이트
3. `POST /api/chicorun/level`: 학습 레벨 변경 및 시작 진도 설정
4. `POST /api/chicorun/reset-achieved-level`: 최고 레벨 기록 초기화
5. `GET /api/chicorun/ranking`: 글로벌 Top 30 랭킹 조회
