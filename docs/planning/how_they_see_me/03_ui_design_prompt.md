# Figma Make 전달용 프롬프트 (한국어 버전 / 복사용)

## 프로젝트 이름
**HowTheySeeMe (HTSM)**

## 목표
React에서 바로 사용할 수 있는 UI 디자인을 생성해주세요.
모든 텍스트는 **영어 + 한국어 다국어(i18n)** 구조를 전제로 만들어야 합니다.

## 중요 조건
- 결과물은 반드시 **React 컴포넌트(JSX)** 형태로 출력
- **Tailwind CSS** 사용
- **반응형 디자인** 필수 (모바일 우선)
- 모든 UI 텍스트는 **하드코딩 금지** → 번역 키 사용
- **i18n 파일 예시**(en / ko) 함께 생성

---

## 1. 다국어(i18n) 규칙 (매우 중요)

화면에 보이는 모든 텍스트는 **번역 키**로 작성해야 합니다.

### 컴포넌트 사용 예시
```jsx
t("hero.title")
t("start.button")
```

### 번역 파일 구조 예시
`src/i18n/en.ts`, `src/i18n/ko.ts`

**예시 (영문)**
```ts
export default {
  hero: {
    title: "See how they see you.",
    subtitle: "Friends help reveal the real you.",
    cta: "Start My Test"
  }
}
```

**예시 (한국어)**
```ts
export default {
  hero: {
    title: "친구들이 보는 나를 확인해보세요.",
    subtitle: "친구의 시선이 진짜 나를 보여줍니다.",
    cta: "테스트 시작하기"
  }
}
```

## 2. 디자인 스타일 가이드

### 스타일 키워드
- 모던
- 깔끔함
- 바이럴 콘텐츠 느낌
- 부드러운 그라데이션
- 둥근 카드 UI
- 친근하고 감성적인 분위기

### 컬러 방향
- **Primary**: 보라 / 인디고 그라데이션
- **Accent**: 파스텔 블루 / 핑크
- **Background**: 밝은 그라데이션 또는 화이트

### 폰트 느낌
- 친근한 Sans-serif
- 큰 Hero 타이포그래피
- 넉넉한 여백

## 3. 반드시 생성해야 하는 페이지

다음 페이지들을 각각 React 페이지 컴포넌트로 생성해주세요.

1. **랜딩 페이지**
2. **테스트 생성 페이지**
3. **공유 페이지** (테스트 만든 사람)
4. **친구 응답 페이지**
5. **결과 페이지** (3D 영역 포함)

---

## 4. 랜딩 페이지 구성

### 포함 섹션
- **Hero 영역**: 큰 헤드라인, 서브 텍스트, CTA 버튼
- **3단계 설명 카드**:
    1. Pick 3 words about yourself
    2. Share with friends
    3. See the real you revealed
- **소셜 프루프 영역**: 결과 카드 썸네일 그리드
- **하단 CTA**: 반복 배치

### 4.1 다국어 옵션 버튼 (Language Switcher)
- **위치**: 전 페이지 공통 헤더의 **우측 상단**
- **이유**: 사용자가 가장 찾기 쉬운 관습적 위치이며, 테스트 중간에도 언제든 언어를 변경할 수 있어야 함 (글로벌/친구 공유 고려)

---

## 5. 테스트 생성 페이지

### 구성 요소
- Progress Header (Step 1 of 2)
- 키워드 Chip 선택 영역 (30~40개)
- 선택 카운터 (Selected: 0 / 3)
- 비활성 → 활성 CTA 버튼

## 8. 결과 페이지 (가장 중요)

### 상단 Hero 영역
**3D Johari Window 캔버스 자리** 확보

**4개 카드 표시**:
- Open Self
- Blind Self
- Hidden Self
- Unknown Self

**각 카드 내용**:
- 영역 이름
- 퍼센트 자리 표시
- 키워드 3개 자리 표시

### 하단 섹션
- 참여 진행 배너
- 공유 유도 영역
- 결과 이미지 다운로드 버튼
- Johari 모델 설명 푸터
