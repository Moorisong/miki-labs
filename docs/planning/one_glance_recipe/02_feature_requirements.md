# 📝 한눈 레시피 - 기능 요구사항 및 구조

## 1. 요리 제목 입력
- 상단 제목 입력칸 배치
- Placeholder: "예: 김치찌개, 제육볶음"
- 미입력 시 기본값: **"나만의 레시피"**

## 2. 재료 입력 영역
### 2.1 입력 방식
- 재료명 직접 입력 (텍스트)
- 수량: + / - 버튼 방식
- 단위: 버튼형 선택 (처음부터 전체 노출)
- "재료 추가" 버튼으로 반복 입력

### 2.2 단위 구성
1. **계량 스푼류**: 큰술, 작은술, 밥숟가락, 티스푼, 한 스푼
2. **무게 단위**: g, kg
3. **부피 단위**: ml, L, 컵, 종이컵
4. **개수 단위**: 개, 알, 장, 줄, 덩이
5. **감각 단위**: 약간, 적당량, 한 꼬집, 한 줌, 조금
6. **기타**: 직접 입력 (선택 시 입력창 활성화)

### 2.3 재료 입력 검증 로직
- 재료명 중복 시 토스트 알림 표시: "이미 추가된 재료입니다. 직접 수정해주세요."
- 중복 시 자동 병합하지 않음

## 3. 조리 단계 구성
### 3.1 단계 기본 구조
- "단계 추가" 버튼
- 최대 20단계 허용
- 순서 변경 가능 (위/아래 이동)
- 조리 단계 0개일 경우 저장 불가

### 3.2 조리 방법 (MVP 최소 구성)
- 끓이기, 볶기, 굽기, 삶기, 튀기기 (추후 확장)

### 3.3 각 단계 구성 요소 및 옵션
1. 조리 방법 선택
2. 사용할 재료 체크 선택
3. 세부 옵션 자동 표시 (기본값 자동 선택)
   - **끓이기**: 불 세기(중불), 시간, 뚜껑 여부(덮고)
   - **볶기**: 불 세기(중불), 기름(식용유), 시간
   - **굽기**: 불 세기, 시간
   - **삶기**: 불 세기, 시간
   - **튀기기**: 기름(식용유), 시간
