# 치코런 학생 프론트엔드 레벨 시스템 Agent Reference

## 02_chicorun_planning.md (레벨 시스템 관련 발췌)

### 7. 🎯 레벨 결정 및 시작 로직 (Option 1)

#### 7.1 목표
레벨 테스트 기능이 없으므로, 학생이 처음 학습을 시작할 때 적절한 레벨에서 시작할 수 있도록 하고, 빠르게 자신의 실력에 맞는 레벨로 조정될 수 있게 한다.

#### 7.2 전체 흐름
1. 학생이 로그인 후 학습 메인 페이지에 진입
2. 시작 레벨 선택 모달 자동으로 표시
3. 학생이 자신의 대략적인 실력을 선택
4. 선택된 범위의 중간 레벨을 `currentLevel`로 설정
5. 학습 시작 후 5~10문제 정도 풀면서 자동 레벨 조정 수행
6. 이후에는 언제든지 상단에서 레벨 변경 가능

#### 7.3 시작 레벨 선택 모달 상세
- **제목**: “시작 레벨을 선택해주세요”
- **설명**: “현재 영어 실력에 맞는 레벨에서 시작하면 더 재미있게 공부할 수 있어요!”
- **선택 옵션**:
    - **완전 초보예요** → 추천 레벨: 1 ~ 15 → 실제 시작 레벨: **8레벨**
    - **기초는 좀 알아요** → 추천 레벨: 11 ~ 35 → 실제 시작 레벨: **23레벨**
    - **중급 정도예요** → 추천 레벨: 31 ~ 60 → 실제 시작 레벨: **45레벨**
    - **꽤 자신 있어요** → 추천 레벨: 56 ~ 85 → 실제 시작 레벨: **70레벨**
    - **고급 수준이에요** → 추천 레벨: 81 ~ 100 → 실제 시작 레벨: **90레벨**
- **UI 요구사항**:
    - 각 옵션은 카드 형태로 시각적으로 구분
    - 선택 시 바로 “시작하기” 버튼 활성화
    - “나중에 선택할게요” 버튼 제공 → 선택 시 기본값으로 1레벨 시작

#### 7.4 자동 레벨 조정 로직 (학습 중 적용)
학습 시작 후 첫 10문제 동안 아래 규칙으로 `currentLevel` 자동 조정:
- **연속 정답 3회 이상** → 레벨 +1
- **연속 정답 5회 이상** → 레벨 +2
- **연속 정답 8회 이상** → 레벨 +3
- **오답 2회 이상 (연속 또는 누적)** → 레벨 -1
- **오답 4회 이상** → 레벨 -2
- **조정 제한**: 최대 ±5레벨까지만 적용 (선택한 시작 레벨 기준)
- **종료 조건**: 10문제 이후에는 자동 조정 종료
- **저장**: 조정된 레벨은 즉시 UI에 반영 + 로컬/서버에 저장

#### 7.5 추가 기능
- 학습 화면 상단에 현재 레벨 표시 (예: "현재 45레벨")
- 옆에 작은 “레벨 변경” 버튼 추가
- “레벨 변경” 버튼 클릭 시 → 시작 레벨 선택 모달 다시 표시
- 모달에서 다시 선택하면 `currentLevel` 즉시 변경 및 학습 문제 재로드

#### 7.6 상태 관리 (Data Schema)
- `student.currentLevel` (number): 현재 학습 중인 레벨 (1~100)
- `student.startLevel` (number): 최초 선택한 레벨 (기록용)
- `adjustmentCount` (number): 자동 조용한 문제 수 (10까지 카운트)

## AI 작업 지침

### 목적
학생이 자신의 실력에 맞는 레벨에서 학습을 시작하고, 학습 성과에 따라 자동으로 레벨이 최적화되도록 구현합니다.

### 작업 단계
1. **상태 정의**:
   - `localStorage` 또는 전역 상태를 사용하여 `currentLevel`, `startLevel`, `adjustmentCount`, `correctCombo`, `incorrectCount` 등을 관리합니다.
2. **시작 레벨 선택 모달 구현**:
   - `ChicorunLevelSelectModal` 컴포넌트를 생성합니다.
   - 5개의 카드 옵션과 '시작하기', '나중에 선택할게요' 버튼을 포함합니다.
   - 최초 진입 시(레벨 정보가 없을 때) 자동으로 띄웁니다.
3. **학습 메인 UI 업데이트**:
   - 헤더 부분에 현재 레벨 표시 및 '레벨 변경' 버튼을 추가합니다.
4. **자동 레벨 조정 로직 구현**:
   - `handleAnswer` 로직에 정답/오답 횟수를 카운트하고, 조건에 따라 `currentLevel`을 증감시키는 로직을 추가합니다.
   - `adjustmentCount`가 10에 도달하면 조정을 중단합니다.
   - 레벨 조정은 `startLevel` 기준 ±5 이내로 제한합니다.
5. **API 연동**:
   - 레벨 변경 시 서버에 `currentLevel`을 업데이트하는 API를 호출하거나 Mock 처리합니다.
   - 레벨 변경 시 문제를 새로 고침합니다.

### 주의사항
- **UX**: 레벨이 자동으로 변경될 때 사용자에게 가벼운 피드백(토스트 등)을 주어 인지하게 합니다.
- **데이터 일관성**: 서버의 `progressIndex`와 `currentLevel`이 동기화되도록 합니다. (1레벨당 100문제)
- **예외 처리**: 레벨은 1 미만이나 100 초과가 되지 않도록 방어 로직을 작성합니다.
