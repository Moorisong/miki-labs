# ⚙️ 한눈 레시피 - 문장 생성 엔진 및 자연스러움 처리

## 1. 개요
문장 생성 엔진은 규칙 기반 문장 품질 관리 시스템입니다. 어색한 조사를 제거하고 반복 표현을 방지하여 사람이 쓴 것 같은 레시피를 생성합니다.

## 2. 엔진 처리 흐름
1. **데이터 정규화**: trim, 연속 공백 제거, 기본값 확정
2. **조사 및 나열 처리**: 마지막 재료에만 조사(을/를) 적용, 3개 이상 시 쉼표 처리
3. **템플릿 선택 및 조건 분기**: 조리 방법과 옵션에 따른 문장 조합
4. **최종 품질 점검**: 반복 표현 체크, 빈 문장 체크

## 3. 문장 자동 처리 로직 (자연스러움)
- **재료 수에 따른 나열**:
  - 1개: `양파를`
  - 2개: `양파와 당근을`
  - 3개 이상: `양파, 당근, 감자를`
- **조사 자동 처리 (받침 여부)**:
  - 받침 있음 → `을` / 받침 없음 → `를`
  - 복합 재료인 경우 마지막 단어 기준
- **옵션 조건 분기**:
  - 시간 미입력 시 `"{시간}간"` 문구 제거
  - 기름 없음 선택 시 `"기름을 두르지 않고"` 처리
  - 불세기 미선택 시 기본값 적용

## 4. 템플릿 기반 구조 (볶기 예시)
- 기본: `팬에 {기름}을 두르고 {불세기}에서 {재료}를 {시간}간 볶아주세요.`
- 옵션(기름 없음): `팬에 기름을 두르지 않고 {불세기}에서 {재료}를 {시간}간 볶아주세요.`

## 5. 최종 품질 점검 (저장 전)
- 동일 종결어(`~해주세요`) 3회 이상 반복 시 템플릿 자동 재생성 또는 연결어 추가
- 조사 이중 적용(`양파를를`) 정규식 자동 제거
- 재료 없는 문장이나 옵션 제거 후 빈 문장 차단 (토스트 알림)
- 종결어 변형: `볶아주세요.`, `볶아줍니다.`, `볶아 주세요.`, `충분히 볶습니다.` 등 다양화
- 연결어 삽입: `이후`, `그리고`, `충분히`, `약간`, `마지막으로` (단계가 많을 때 선택적 삽입)
