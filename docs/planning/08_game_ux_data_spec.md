# 🎮 인형뽑기 게임 UX & 데이터 기획서

---

## 1. 기획 핵심 원칙

- 로그인 없이도 게임 플레이 가능
- 로그인은 **선택**, 랭킹 기록은 **보상**
- 시도 횟수와 쿨타임 규칙은 로그인/비로그인 동일
- 유저에게 불리하게 느껴지는 리셋 또는 차별 없음

---

## 2. 로그인 / 비로그인 정책

### 2-1. 비로그인 상태
- 게임 플레이 가능
- 시도 횟수 및 쿨타임 정상 적용
- 랭킹 기록 저장 불가

### 2-2. 로그인 상태
- 게임 플레이 가능 (동일)
- 성공 기록 및 점수 랭킹 저장 가능

> 로그인은 **게임 플레이 권한이 아닌, 기록 저장 권한을 제공**한다.

---

## 3. UX 문구 기획

### 3-1. 비로그인 기본 안내

#### 노출 위치
- 게임 메인 화면 상단  
- 또는 시도 횟수 표시 영역 인근

#### 노출 조건
- 비로그인 상태에서 상시 노출

#### 문구
- **"로그인 없이도 플레이할 수 있어요"**
- **"로그인하면 기록을 랭킹에 저장할 수 있어요"**

---

### 3-2. 로그인 유도 문구 (CTA)

#### 노출 위치
- 로그인 버튼 영역
- 랭킹 메뉴 접근 전 화면

#### 노출 조건
- 비로그인 상태일 때만 노출

#### 문구
- **"지금 로그인하고 랭킹에 도전하세요!"**

---

### 3-3. 게임 성공 시 로그인 유도 (비로그인)

#### 노출 위치
- 성공 연출 후 결과 팝업 또는 토스트

#### 노출 조건
- 비로그인 상태 + 게임 성공 시

#### 문구
- "🎉 성공! 로그인하면 이 기록을 랭킹에 저장할 수 있어요"

---

## 4. 시도 횟수 기획

### 4-1. 기본 규칙
- 기본 시도 횟수: **3회**
- 실패 시: 시도 횟수 -1
- 성공 시: 시도 횟수 +1
- 시도 횟수는 0 미만으로 내려가지 않음

---

### 4-2. 시도 횟수 표시 UX

#### 노출 위치
- 게임 화면 상단 HUD (상시 고정)

#### 표시 예시
- "남은 시도 횟수: 3회"
- "남은 시도 횟수: 1회"

---

### 4-3. 시도 횟수 1회 UX 강화 (선택)

#### 노출 조건
- 남은 시도 횟수 = 1

#### 보조 문구
- "마지막 기회예요!"
- "성공하면 기회가 하나 더 늘어나요!"

---

## 5. 쿨타임 기획

### 5-1. 쿨타임 발생 조건
- **시도 횟수가 0이 되었을 때만** 쿨타임 시작
- 쿨타임 시간: **1시간**

---

### 5-2. 쿨타임 UX 표시

#### 노출 위치
- 시도 횟수 영역 대체 또는 게임 화면 중앙

#### 표시 내용
- "시도 횟수 소진"
- "다음 시도까지 남은 시간: 00:59:59"

#### 보조 문구
- **"쿨타임 종료 후 시도 횟수 3회가 충전돼요"**

---

### 5-3. 쿨타임 중 제어
- 게임 시작 버튼 비활성화
- 비활성 상태 시각적으로 명확히 표현

---

### 5-4. 쿨타임 종료 UX
- 시도 횟수 자동 3회 충전
- 문구:
  - "🎉 시도 횟수 3회가 충전되었어요!"
  - "다시 도전해보세요!"

---

## 6. 데이터 저장 정책 (중요)

### 6-1. 저장 원칙 요약
> **플레이 제한은 디바이스 기준으로 통일하고,  
계정(DB)은 기록 저장 용도로만 사용한다.**

---

### 6-2. 클라이언트 스토리지 저장 대상 (로그인/비로그인 공통)

- 남은 시도 횟수
- 쿨타임 종료 시각

#### 저장 위치
- Web: `localStorage` 또는 `IndexedDB`

#### 예시 구조
```json
{
  "remainingAttempts": 1,
  "cooldownEndAt": 1700000000000
}
```

#### 특징
- 로그인 여부와 무관하게 동일 로직 적용
- 로그인 전/후 상태 변화 없음

---

### 6-3. 서버(DB) 저장 대상 (로그인 유저만)
- 유저 ID
- 성공 기록
- 점수 / 랭킹 데이터
- 최고 기록

#### 예시 구조
```json
{
  "userId": "user_123",
  "score": 150,
  "createdAt": "2026-01-21T12:00:00Z"
}
```

---

### 6-4. 로그인 시 처리 규칙
- 로그인 시 시도 횟수 및 쿨타임 상태는 변경하지 않음
- 스토리지 ↔ DB 간 상태 동기화 없음
- 로그인은 기록 저장 권한만 부여
