# 치코런 학생 프론트엔드 에이전트 (Frontend Student Agent) 지침서

## 🎯 목표
치코런 학생용 학습 페이지 및 플레이 엔진부(게임 플로우)를 독립적으로 구현합니다. 교사용 관리/꾸미기 요소와 의존도를 최소화하여 독립 작동 가능해야 합니다.

## 📝 작업 내용

### 1. 학생 로그인/재접속 플로우 구현
- 랜딩 화면에서 클래스코드 + 닉네임 + 비밀번호 입력 폼 구성. (3초 내 시작)
- 로그인 성공 시 발급받은 JWT를 로컬/상태에 저장 (Stateless 통신).

### 2. 학습 플레이 시스템 
- **문제 UI**: 상단(레벨, 번호, 포인트), 중앙(객관식 문장), 하단(선택지 4개).
- **정답 처리 (MZ 피드백)**:
  - 오답 시 빨간 효과 + '1~2줄 짧은 핵심 해설' 노출. 다음 문제 대기 안 하고 다시 시도.
  - 정답 시 초록 효과. 즉시 다음 문제로 자동 이동.
- **연속 정답(Comobo) UX**: 
  - 3연속, 5연속, 10연속(퍼펙트) 시 화면 중앙 콤보 연출 애니메이션 추가 (10연속 시 +20P).
- **포인트 0점 상황 연출**: 3회 시도 후 정답 시 포인트 0점에 대한 안내 및 위로 이모지 노출.

### 3. 엔딩 및 복구 로직
- **레벨 클리어 (1~100레벨 구간)**: 명언 표출과 다음 레벨 진행 버튼 컴포넌트 구현.
- **최종 엔딩(Level 100 달성)**: 축하 화면 + '1레벨부터 다시' (progress 초기화, 포인트 유지) 액션.
- 서버에서 던전 복구 (재접속) 시, 저장된 `seed`에 맞게 이전 문제 위치로 돌려넣음.

## ⚠️ 주의사항 (병렬성 확보)
- **API 연동 최소화**: REST API가 아직 연동되지 않았다면 Mock 함수를 생성해 렌더링/인터랙션/이펙트를 먼저 완성하세요.
- **꾸미기 컴포넌트 생략**: 다른 에이전트가 `ProfileCard` 등을 개발하기 전까진 그 자리를 더미 `div`로 남겨둡니다.
