# Haroo Box Agent Reference

## Haroo Box Planning (01_haroo_box_planning.md)
# 📦 하루상자 (HaruBox) 기획 개요

## 1. 서비스 컨셉
하루상자는 짧은 시간 안에 가볍게 즐길 수 있는 **아기자기한 놀이·게임·기록 콘텐츠를 모아둔 웹 플랫폼**입니다.

사용자는 하루상자 안에서:
- 게임을 즐기고
- 소소한 기록을 남기고
- 결과물을 저장하거나 공유하며
하루의 작은 재미를 수집합니다.

---

## 2. 핵심 방향성
- **인형뽑기(구 뽑기중독)**는 메인이 아닌 **대표 콘텐츠 중 하나**
- 로그인은 **선택적**
- 로그인 없이도 대부분의 콘텐츠 체험 가능
- 로그인 시 **기록·저장·개인화** 제공

---

## 3. 로그인 설계

### 3.1 로그인 원칙
- **초기 진입 허들은 낮게**
- "로그인 안 해도 됨" (Guest Play First)
- 단, 저장/히스토리/랭킹 등록은 로그인 필요

### 3.2 로그인 방식
- 소셜 로그인 위주 (Google, Kakao)
- 이메일/비밀번호 방식은 선택사항

### 3.3 로그인 시 제공 기능
- 플레이 기록 저장
- 개인 보관함 (이미지, 결과물)
- 콘텐츠별 통계
- 기기 변경 시 데이터 유지

---

## 4. 전체 페이지 구성

### 4.1 메인 페이지 (`/`)
**목적:** 하루상자의 분위기 전달 및 콘텐츠 탐색

*   **상단**:
    *   로고: 하루상자
    *   메뉴: 콘텐츠 | 로그인
*   **히어로 섹션**:
    *   문구: "오늘의 작은 재미를 담다"
*   **콘텐츠 카드 그리드**:
    *   인형뽑기 (Claw Machine)
    *   미니게임 (Coming Soon)
    *   편지 + 스티커 (Coming Soon)
*   **하단**:
    *   서비스 설명
    *   SEO 텍스트

### 4.2 콘텐츠 목록 (`/contents` - 예정)
**목적:** 제공 중인 모든 놀이 콘텐츠 모음 (현재 메인 페이지에서 바로 접근 가능하도록 구현)

*   카드형 UI
*   상태 표시: NEW, 인기, 준비중
*   클릭 시 각 콘텐츠 상세/플레이 페이지로 이동

### 4.3 인형뽑기 (`/claw`)
*   **URL**: `/claw` (이전 `/` 및 `/game` 기능 통합)
*   **기능**:
    *   3D 인형뽑기 게임 (Game Logic Refer: `docs/planning/claw_machine/product_spec.md`)
    *   물리엔진 기반 조작
    *   결과 이미지 저장 가능 (로그인 시)
    *   랭킹 시스템

### 4.5 마이페이지 (`/mypage`)
*   로그인 정보
*   콘텐츠 이용 통계
*   데이터 초기화
*   탈퇴

## 6. 확장 고려 사항
- 콘텐츠 추가 시 `/contents/{slug}` 또는 `/{slug}` 구조 유연하게 유지 (현재 `/claw` 사용)
- 모든 콘텐츠는 "하루상자" 세계관과 디자인 톤앤매너 공유
- 추후 모바일 앱(WebView 기반 등) 확장 가능 구조

## 7. 한 줄 요약
하루상자는 **하루의 소소한 재미를 담아두는 작은 놀이 상자**입니다.

## AI 작업 지침

### 목적
하루상자 플랫폼의 핵심 가치와 아키텍처를 유지하며, 새로운 콘텐츠 추가 및 기존 콘텐츠(인형뽑기 등)의 통합을 원활하게 수행한다.

### 작업 단계
1.  **플랫폼 일관성 유지**:
    - 모든 페이지는 `Haroo Box`의 디자인 시스템(톤앤매너)을 따른다.
    - 네비게이션 바와 푸터는 전역적으로 일관되게 유지한다.
2.  **콘텐츠 확장성 고려**:
    - 새로운 게임이나 기능 추가 시 `/app/[content-slug]` 형태의 디렉토리 구조를 준수한다.
    - 공통 컴포넌트(모달, 버튼 등)는 `client/components`에서 재사용한다.
3.  **로그인 경험 최적화**:
    - 비로그인 유저에게 강제적인 로그인을 요구하지 않는다.
    - 기능 제한이 필요한 경우(예: 저장, 랭킹)에만 부드럽게 로그인을 유도한다.

### 주의사항
- **데이터 보존**: 기존 `Claw Addict`의 랭킹 데이터나 사용자 정보는 손실되지 않도록 마이그레이션 및 호환성을 유지한다.
- **모바일 대응**: 반응형 디자인을 최우선으로 고려하며, 모바일 터치 환경에서의 사용성을 해치지 않는다.
- **과도한 복잡성 지양**: "가볍게 즐기는" 컨셉에 맞게, UI/UX는 직관적이고 단순해야 한다.
