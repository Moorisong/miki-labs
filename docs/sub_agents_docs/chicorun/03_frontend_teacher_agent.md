# 치코런 교사 대시보드 프론트엔드 에이전트 (Frontend Teacher Agent) 지침서

## 🎯 목표
하루상자 로그인 기반 교사용 치코런 관리자 화면 및 공통 랭킹 보드를 담당합니다. 학생의 게임 데이터나 플레이 로직이 구현되지 않았더라도 모의(모크) 데이터로 우선 병렬 레이아웃 작업을 마칩니다.

## 📝 작업 내용

### 1. 클래스 관리 대시보드 (Teacher Only)
- 교사 소유의 `classes` 리스트 출력.
- [클래스 생성] 버튼 구현 (Unique Code 모달 팝업 및 Confirm).
- 각 개별 클래스의 코드 복사 기능(Clipboard) 구현.

### 2. 학생 관리 테이블 (Teacher Only)
- 해당 클래스에 속한 `students` 리스트 데이터 호출 및 렌더.
- 데이터 필드: `nickname`, `level(계산된)`, 학생 프로필 카드.
- **중요 액션부**: [비번 초기화(0000)], [닉네임 변경 폼] 포함 (실수 방지를 위해 Confirm 모달 연계).

### 3. 공통 랭킹 보드 (Teacher & Student)
- 클래스 내 포인트 순 석차를 보여주는 테이블 구현.
- `nickname`, 포인트, 프로필 카드 (`badge + nickname style`)만 노출 (진도 progress는 가림 처리).

## ⚠️ 주의사항 (병렬 및 모듈 분리)
- 랭킹 보드나 학생 리스트에서 출력하는 학생의 `ProfileCard`(뱃지, 스티커 조합) 컴포넌트는 커스터마이징 에이전특가 구현합니다. 
- 임시로 `<DummyProfile nickname={...} badge={...} />` 등을 이용해 UI 규격만 잡아두세요. API가 아직 없으면 더미 리스트 배열을 만들어 확인합니다.
