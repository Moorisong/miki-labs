### 🎮 치코런(Chicorun) - 워드 러시(Word Rush) 기획서

---

# 📌 1. 게임 개요
- **워드 러시 (Word Rush)**는 영어 설명(힌트)을 보고 해당 단어를 직접 타이핑하는 **속도 + 정확도 기반 타이핑 게임**이다.
- 제한 시간 내 최대 정답 수 기록
- 의미(mid) 단위 문제 출제
- 복수 정답(en 배열) 허용
- 레전드 기록 경쟁 구조

---

# 💰 2. 포인트 시스템 UI/UX

## 2.1 기본 원칙
- rankPoint와 coin은 반드시 분리 표시
- 점수(경쟁)와 재화(소비)를 명확히 구분

## 2.2 상단 HUD
[ 🏆 12,450 ]   [ 💰 3,200 ]
  rankPoint         coin

## 2.3 UX 규칙
- rankPoint → 강조 (크고 굵게)
- coin → 보조 (작고 서브 느낌)

## 2.4 게임 종료 보상 UI
🏆 +150
💰 +45
- 카운트업 애니메이션 적용
- coin은 약간 늦게 증가

## 2.5 소비 UI
- 펫 구매 시 coin 사용
- 감소 애니메이션 제공

## 2.6 랭킹 UI
- rankPoint만 표시
- coin은 표시하지 않음

## 2.7 색상 규칙
- rankPoint: 골드/노랑
- coin: 초록/블루

---

# 🧠 3. 핵심 플레이 방식
1. 영어 설명(wordHints.text) 제공
2. 유저가 영어 단어 입력
3. 정답 체크: `input ∈ meaning.en`
4. 정답 시 다음 문제 진행

---

# ⏱️ 4. 게임 구조
- 최대 30문제
- 제한 시간 기반 진행
- 시간 종료 시 결과 집계

---

# 📦 5. 문제 생성 시스템 (Word System 기반)

## 5.1 사용 데이터
- words
- wordHints
- questionTypes (qid)

## 5.2 문제 구조
{
  qid: 3,
  wid: Number,
  mid: Number,
  hintId: ObjectId,
  hint: String
}

## 5.3 생성 흐름
1. level 기반 단어 조회
2. wid 선택
3. mid 선택
4. wordHints 조회
5. difficulty 필터 적용
6. 랜덤 힌트 선택

## 5.4 정답 판정
`input ∈ meaning.en`
- 복수 정답 허용
- 대소문자 무시
- 공백/특수문자 정규화 필요

---

# 🔁 6. 중복 방지 시스템
- 기준: `wid + mid + hintId`
- 추가 전략: recent wid 제한, mid 연속 방지
- 세션 구조:
{
  used: [ { wid, mid, hintId } ],
  expiresAt: timestamp
}

---

# 🎯 7. 점수 시스템
- **기본 점수**: 정답 수 기반
- **콤보**: 연속 정답 시 증가, 오답 시 초기화
- **점수 요소**: 정답 수, 콤보, 속도

---

# 🏆 8. 레전드 시스템
- 최고 기록 유저 = 레전드
- 동작: 결과 기록 생성 -> 레전드 기록과 비교 -> 초과 시 교체
- 연출: 펫 vs 펫 전투, 레전드 자동 공격

---

# 🐾 9. 펫 & 스킨 시스템
- **펫**: 실력 보조만 수행 (오타 보정, 콤보 유지 보조, 시간 완화, 힌트 제공)
- **스킨**: 소유 및 장착 시스템 동일

---

# 💰 10. 포인트 시스템 & 11. 학생 스키마 업데이트
- `rankPoint`: 랭킹 기준, 차감 없음
- `coin`: 소비 재화, 펫 구매 사용
- 스키마에 `rankPoint`, `coin`, 펫 및 스킨 데이터 추가됨

---

# ⚔️ 12. 전투 연출 시스템
- 구조: 좌(유저 펫), 우(레전드 펫), 중앙(이펙트)
- 상태 흐름: 대기 → 공격 → 피격 → 결과
- 구현: CSS 애니메이션, Sprite, 흔들림 이펙트

---

# ⚙️ 13. API 구조
1. **START**: `POST /game/start` (문제 배열 응답)
2. **END**: `POST /game/end` (결과 제출)
- 서버 측 치팅 방지 로직 (wid/mid/hint 검증, 입력 속도 체크)
