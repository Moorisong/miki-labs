# 치코런 꾸미기 전담 프론트엔드 에이전트 (Frontend Customizing Agent) 지침서

## 🎯 목표
사용자의 닉네임, 뱃지, 카드의 꾸미기(스타일, 스티커 배치, 드래그 드롭) 기능만 완전히 격리하여 독립 구현합니다. 이미지 업로드 없이 메타데이터(JSON) 기반 브라우저 렌더링에 집중합니다.

## 📝 작업 내용

### 1. 프로필 카드(ProfileCard) 통합 컴포넌트 렌더러
- 서버에서 넘어오는 `cardStyle`, `badgeId`, `frameId`, `stickers`, `nicknameStyle` 메타객체(JSON)를 읽어와 하나의 카드 화면에 합성(Layering)하여 그리는 공통 컴포넌트(`ProfileCard`)를 작성합니다.
- CSS `position: absolute`, `transform(scale, rotate)`, 그리고 `color` 등의 인라인/동적 스타일링 지원.
- 다른 에이전트들이 공통 코드로 사용할 수 있도록, 외부와의 의존성이 없이 `props={...}`만 받으면 렌더되게끔 독립(pure) 컴포넌트로 작성할 것.

### 2. 꾸미기 상점 에디터 UI
- **닉네임 스타일러**: Color Picker 폰트, bold, italic, underline 버튼 UI.
- **스티커 & 프레임 D&D 기능**: 
  - 사용자가 스티커를 누르고 x, y 좌표로 드래그하여 배치할 수 있는 유저 반응형 캔버스 뷰(`react-draggable` 같은 훅, 혹은 Custom 이벤트 사용 고려)
  - 저장 시 위치 정보를 `x, y, scale, rotate` 데이터로 직렬화하여 상태 저장.
  - 커스터마이징 횟수 당 비용 차감(1회: 100P, 2회: 200P 등 로직 표출).

## ⚠️ 주의사항 (토큰 최소화)
- 학습 로직이나 관리자 로직의 구현 코드는 전부 무시하세요. 백엔드 연동을 기다리지 않습니다.
- 꾸미기 설정(`props`)을 바꿔가면서 D&D와 렌더링 성능 지연 없이 자연스러운 조작감이 나오는지가 최우선 검증 대상입니다.
