# 🎯 뽑기중독 (Ppopgi Addict) - 제품 기획서 (Product Spec)

## 1. 프로젝트 개요 (Overview)
* **서비스명**: 뽑기중독 (Ppopgi Addict)
* **Repository**: `ppopgi-addict`
* **핵심 목표**: 웹에서 실제 인형뽑기의 조작감과 재미를 최대한 재현
* **체감 기준**: 물리 엔진(80%) + 연출 및 확률 조작(20%)의 조화
* **초기 범위**: 핵심 게임 플레이 + 로그인 + 랭킹 시스템
* **기타**: SEO 고려 (검색 및 공유 시 노출 최적화)

---

## 2. 리얼리즘 구현 철학 (Realism Philosophy)
이 프로젝트는 단순한 웹 게임이 아니라, "진짜 인형뽑기를 조작하고 있다는 착각"을 주는 것을 목표로 합니다.

### 2.1 핵심 목표
> 플레이어가 느끼기에 **"내가 방금 저렇게 움직였기 때문에 결과가 이렇게 나왔다"**라는 인과관계가 명확하게 체감되는 게임

* **Random vs Calculation**: 랜덤처럼 보이지만, 실제로는 **수많은 계산의 결과**여야 합니다.
* **Micro-Control**: 아주 미세한 조작 차이로도 결과가 달라질 수 있어야 합니다.
* **Failure Emotion**: 실패했을 때 "사기"가 아니라 **"아, 내가 놓쳤다"**라는 감정을 유도해야 합니다.

### 2.2 기본 철학: 80% 물리 + 20% 연출
* 물리는 최대한 실제에 가깝게 계산합니다.
* 연출과 확률은 **결과를 조정하는 보조 수단**으로만 사용합니다.
* 절대 "버튼 누르면 성공/실패" 구조로 만들지 않습니다.
> **결과는 항상 과정의 부산물이어야 합니다.**

### 2.3 미세 조작이 결과에 영향을 주는 구조
* **위치 오차**: 집게 중심이 인형 무게중심에서 벗어난 정도 (±1~2cm 차이 반영)
* **타이밍 오차**: 집게 하강/닫힘/상승 타이밍에 따른 결과 변화

### 2.4 집게 및 환경 리얼리즘
* **집게**: 3갈래이며, 각 갈래의 힘이 미세하게 다르며 흔들림(관성)이 존재합니다.
* **인형**: 인형마다 **무게, 마찰, 형태(Collider)**가 다르며, 체감 난이도가 달라집니다.
* **환경**: 바닥 마찰, 출구 턱 등 환경 요소가 성공 확률에 개입합니다.

## 3. 게임 로직 및 메카닉 (Game Mechanics)

### 3.1 구현 원칙
* **목표**: 100% 물리 시뮬레이션보다는 "재미있는 조작감" 우선
* **React 연동**: 
  - React는 UI 오버레이(점수, 버튼, HUD) 관리
  - 실제 게임 물리 엔진 및 상태는 `Zustand` 스토어와 `cannon-es` 물리 엔진이 연동되어 관리

### 3.2 조작 및 입력 (Controls)
* **키보드 컨트롤**: 한국어 입력 상태에서도 오동작하지 않도록 `event.code` 방식 사용
    * **이동**: `W` / `A` / `S` / `D` 또는 방향키 (`↑`, `←`, `↓`, `→`)
    * **동작**: `Space` 또는 `Enter` (게임 시작 / 집게 하강)

### 3.3 게임 규칙 (Game Rules)

#### 시도 횟수 (Attempts)
* **기본 제공**: 5회
* **차감 규칙**:
    * **실패 시**: 시도 횟수 1회 차감 (-1)
    * **성공 시**: 시도 횟수 차감 없음 (성공 보너스로 1회 충전 후 1회 소모 = Net 0)
    * **의도**: 성공하면 계속해서 플레이할 수 있는 "무한 도전" 기회 제공
* **저장 위치**: 브라우저 `localStorage` (로그인 여부와 무관하게 디바이스 기준)

#### 쿨타임 (Cooldown)
* **발동 조건**: 시도 횟수가 0이 되었을 때
* **대기 시간**: 1시간 (60분)
* **해제 보상**: 시도 횟수 5회 충전
* **표시**: HUD 상단에 남은 시간 표시

#### 로그인 정책 (Auth Policy)
* **기본 원칙**: 
    * 로그인 없이도 **게임 플레이 가능**
    * 로그인은 **기록(랭킹) 저장 권한**을 위한 선택 사항
* **비로그인 유저**: 플레이 가능, 랭킹 등록 불가 (게임 종료 후 안내)
* **로그인 유저**: 플레이 결과가 랭킹 서버(DB)에 기록됨

### 3.4 상세 메카닉 (Mechanics)

#### 집게 (Claw)
* **구조**: 3갈래 집게 (Three-pronged)
* **물리**:
  - `cannon-es`를 사용한 물리 연산
  - `visualClawPosition`을 사용하여 물리 엔진 위치와 렌더링 위치 동기화
  - **안전 장치**: 집게가 완전히 복귀하지 않은 상태 (`y < 3.8`)에서는 하강 명령 무시

#### 게임 플레이 흐름 (Flow)
1. **대기 (Idle)**: 시작 대기. 비로그인 유저에게 로그인 유도 문구 노출.
2. **이동 (Moving)**: 유저 입력에 따라 집게 이동 (속도 및 가속도 적용).
3. **하강 (Dropping)**: 버튼 입력 시 하강 시작.
4. **잡기 (Grabbing)**: 충돌 감지 시 그립력 적용. 물리 엔진을 통해 인형을 잡음.
5. **상승 (Rising)**: 집게 상승.
6. **복귀 (Returning)**: 시작 위치(배출구 위)로 이동. 흔들림(Spring) 물리 적용.
7. **성공/실패 판정**:
   - 배출구(Hole) 통과 시: **성공** (Score 증가, Attempt 유지)
   - 도중 낙하 또는 빈손: **실패** (Attempt 차감)

### 3.5 카메라 (Camera)
* 집게의 위치를 추적하되 `damp` 로직을 적용하여 부드러운 움직임 구현
* 쿼터뷰 시점 고정
