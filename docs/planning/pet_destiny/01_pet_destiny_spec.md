# 운명연구소 (Animal Destiny Lab) 상세 기획 및 명세

## 1. 콘텐츠 개요

운명연구소는 반려동물과 집사의 관계·성향·운세를 동양 명리학의 **오행 이론**을 기반으로 재미있게 풀어보는 콘텐츠이다.
본 서비스는 **엔터테인먼트 목적의 운세 콘텐츠**이며 AI API 없이 **룰 기반 계산 시스템**으로 구현한다.

---

## 2. 서비스 목표

### 핵심 목표
- 로그인 없이 가볍게 즐기는 콘텐츠
- 카카오 공유하기 쉬운 결과 생성
- 로그인 유도 없음
- SEO 유입 강화 콘텐츠

### 콘텐츠 포지션
하루상자의 **대표 콘텐츠 시리즈**로 확장 가능한 구조

---

## 3. 핵심 방향성

### 3.1 AI 사용하지 않음
- 외부 AI API 사용 ❌
- 룰 기반 운세 시스템 ⭕
- 운세 서비스의 본질 → **계산 + 매핑 + 템플릿**

### 3.2 결정적 결과 시스템 (Deterministic)
- 같은 입력 → 항상 같은 결과 출력
- **입력값**: 집사 생년월일, 반려동물 생년월일, 동물 종류
- **출력값**: 성향 분석, 궁합 점수, 건강 운, 마음 경계 지수, 평생 운 흐름, 올해 운세, 결과 문장
- **문장 선택**: **Seed 기반 방식** 사용

---

## 4. 명리학 적용 범위

### 정통 사주 vs 본 서비스
- **정통 사주**: 생년월일시 사용, 절기 계산, 월주/일주/시주 포함
- **본 서비스**: 연도 중심 사주, 천간 + 오행 중심 분석
- → **간략 명리학 기반 운세 콘텐츠**

> **서비스 안내 문구**: "동양 명리학의 오행 이론을 바탕으로 재미있게 풀어본 반려동물 운명 콘텐츠입니다."

---

## 5. 핵심 알고리즘

### 5.1 띠 계산
`animalIndex = (year - 4) % 12`
- **띠 순서**: 쥐 소 호랑이 토끼 용 뱀 말 양 원숭이 닭 개 돼지

### 5.2 천간 계산
`stemIndex = (year - 4) % 10`
- **천간 순서**: 갑 을 병 정 무 기 경 신 임 계

### 5.3 천간 → 오행 변환
- 갑 을 → 목
- 병 정 → 화
- 무 기 → 토
- 경 신 → 금
- 임 계 → 수
- **결과** → **오행 타입 생성**

---

## 6. 오행 타입 성격

- **목 🌳**: 호기심 많음, 활동량 많음, 사고 발생률 높음
- **화 🔥**: 애교 많음, 감정 표현 강함, 관심 욕구 높음
- **토 🪨**: 안정적, 잠 많음, 루틴 선호
- **금 ⚙️**: 깔끔함, 예민함, 프라이드 높음
- **수 💧**: 독립적, 관찰형, 속마음 알기 어려움

---

## 7. 집사–반려동물 궁합 계산

**오행 순환**: 목 → 화 → 토 → 금 → 수 → 목

### 점수 규칙
- 상생 +2
- 동일 +1
- 상극 -2

### 궁합 결과
- **+3 이상**: 운명 공동체
- **+1 ~ +2**: 찰떡 파트너
- **0**: 평범한 집사
- **-1 ~ -2**: 애증 관계
- **-3 이하**: 주인 교체 고려

---

## 8. 평생 건강 운 (핵심 콘텐츠)

**오행 ↔ 취약 건강 포인트 매핑**
- **목 🌳**: 간 / 눈 / 활동성 사고
- **화 🔥**: 심장 / 혈관 / 과흥분
- **토 🪨**: 소화 / 체중 / 비만
- **금 ⚙️**: 폐 / 피부 / 알러지
- **수 💧**: 신장 / 비뇨 / 노화

> **결과 예시 문장**: “평생 건강 관리 포인트를 알려주는 콘텐츠”

---

## 9. 마음 경계 지수 (감정 콘텐츠)

입양·길생활 경험 집사 공감 콘텐츠. 오행 기반 심리 성향 분석.

- **목 🌳** → 낯가림 적음
- **화 🔥** → 애착 강함
- **토 🪨** → 신뢰 형성 느림
- **금 ⚙️** → 경계심 높음
- **수 💧** → 독립성 강함

**표현 원칙**:
- “유기 경험” 직접 언급 ❌
- “마음을 여는 속도” 형태로 표현 ⭕

---

## 10. 평생 운 흐름

나이 기반 라이프 사이클
- **0~2세**: 성장운
- **3~6세**: 전성기
- **7~10세**: 안정기
- **11세+**: 노현자 운

---

## 11. 올해 운세 계산 (상세 업데이트)

반려동물 오행 vs 올해 오행 비교 및 Seed 기반 점수제를 적용하여 풍부한 결과를 제공한다.

### 11.1 올해 운세 알고리즘
1. **올해 오행 도출**: `(currentYear - 4) % 10` → 천간 → 오행 (`currentElement`)
2. **기본 점수 계산**:
   - 상생: +2점
   - 동일: +1점
   - 상극: -1점
3. **Seed 가중치 적용**: `scoreAdjust = (seed % 3) - 1` (범위: -1 ~ +1)
4. **최종 점수**: `finalScore = baseScore + scoreAdjust`

### 11.2 결과 레이블 및 이모지 매핑
| 점수 (Final Score) | 운세 레이블 | 이모지 | 의미 |
|---|---|---|---|
| **+3 이상** | **대길** | 🍀 | 최고의 운세, 기회가 찾아옵니다. |
| **+1 ~ +2** | **길** | 🌸 | 좋은 기운이 함께합니다. |
| **0** | **보통** | 🌤️ | 평범한 한 해가 될 거예요. |
| **-1 ~ -2** | **주의** | ⚠️ | 건강과 감정 관리가 필요해요. |
| **-3 이하** | **흉** | 🌩️ | 예상치 못한 어려움이 있을 수 있어요. |

### 11.3 결과 문장 구성 규칙
결과 `yearFortune` 값은 단순 레이블이 아닌 **전체 문장**으로 완성하여 반환한다.

**포맷**: `[레이블] [이모지] [이유 설명]. [상세 설명]`

**예시 데이터**:
- **대길**: "대길 🍀 올해는 목(木) 기운이 서로를 돕습니다. 최고의 기회가 찾아올 거예요!"
- **길**: "길 🌸 화(火) 기운이 조화를 이룹니다. 좋은 기운이 함께합니다."
- **보통**: "보통 🌤️ 올해는 무해무득한 시기입니다. 평범한 한 해가 예상됩니다."
- **주의**: "주의 ⚠️ 금(金) 기운이 충돌할 수 있습니다. 건강과 감정 관리가 필요해요."

**공유 텍스트(`shareText`) 및 결과 화면(`yearFortune`)에 동일하게 적용한다.**

## 12. 결과 편향 개선 시스템

> **목적**: 사용자가 항상 긍정적 결과만 보지 않도록 조정하여 결과 신뢰도와 재미 강화, 공유 콘텐츠 다양성 확보

### 12.1 궁합 점수 Seed 기반 가중치 변동

기존 오행 관계 점수에 Seed 기반 미세 조정을 적용한다.

**계산식**:
```js
seedAdjust = (seed % 3) - 1;  // -1, 0, +1 중 하나
finalScore = baseScore + seedAdjust;
```

**결과 레이블 구간 (최종)**:
| 점수 | 레이블 | 이모지 | 설명 |
|------|--------|--------|------|
| 90~100 | 천생연분 | 💕 | 천생연분이에요! |
| 80~89 | 환상궁합 | ✨ | 환상의 궁합이에요! |
| 70~79 | 좋은인연 | 🌟 | 좋은 인연이에요! |
| 60~69 | 이해필요 | 💗 | 서로를 이해하는 시간이 필요해요! |
| 50~59 | 노력필요 | 💪 | 함께 노력하면 좋아질 거예요! |
| 40~49 | 애증관계 | 🌀 | 밀당이 필요한 사이예요! |
| 0~39 | 주인교체 | ⚠️ | 인내심을 갖고 천천히 다가가세요! |

### 12.2 문장 템플릿 다양화

점수 구간별로 **긍정 / 중립 / 경고 / 부정** 문장 풀을 구성한다.

**예시 문장 풀**:
- **긍정 (80+)**: "운명 공동체 💛", "찰떡 파트너 ✨"
- **중립 (60~79)**: "서로 이해하며 지내는 평범한 관계 🐾", "나쁘지 않은 사이예요 🌤️"
- **경고 (40~59)**: "조금 신경 쓸 필요가 있어요 ⚠️", "서로의 시간이 필요해요 🕐"
- **부정 (0~39)**: "애증 관계 가능성 있음 💢", "함께 노력이 많이 필요해요 💦"

**선택 방식**:
- Seed 기반 deterministic 선택 유지
- `index = seed % categoryMessages.length`

### 12.3 건강 운 및 올해 운세 변동

Seed 기반으로 건강 포인트 및 올해 운세에 변동 적용:

**건강 운 변동**:
```js
healthVariant = seed % 2;  // 0 또는 1
// 예: 목(木) → 0: "간 건강 주의", 1: "눈 건강 주의"
```

**올해 운세 변동**:
- 기본 오행 관계에 추가 표현 변동
- summary, shareText에도 반영하여 결과 균형 유지

### 12.4 전반적 균형 원칙

- Seed 기반 deterministic 특성 유지
- 각 계산 단계에 **±1 범위 변동** 적용
- 최종 결과 분포 목표:
  - **긍정 결과**: 약 40%
  - **중립 결과**: 약 35%
  - **주의/경고 결과**: 약 25%

# 페이지 구조 (IA)

## 1. 개요
운명연구소는 하루상자 메인 페이지에서 진입하는 **2페이지 구조의 콘텐츠 플로우**로 구성한다.
본 콘텐츠는 **짧은 입력 → 즉시 결과 → 공유** 흐름을 최적화하는 것을 목표로 한다.

## 2. 사용자 흐름

**사용자 이동 경로**:
`/` (하루상자 메인) → `/pet-destiny` (입력 페이지) → `/pet-destiny/result?seed={hash}` (결과 페이지)

**흐름 요약**:
1. 메인 페이지에서 콘텐츠 카드 클릭
2. 반려동물 정보 입력
3. 결과 확인 및 공유

**예외 처리**:
- 결과 데이터 없으면 입력 페이지로 리다이렉트(중요). `localStorage` 결과 없으면 → `/pet-destiny` 이동. API 직접 호출 방지용 404 처리.

---

## 3. 입력 페이지

### 3.1 URL
`/pet-destiny`

### 3.2 페이지 역할
- 콘텐츠 소개
- 사용자 입력 수집
- 결과 페이지로 이동

### 3.3 페이지 구성
- **타이틀**: 운명연구소 🔮
- **서브 카피**: 생년월일로 보는 반려동물 사주 & 궁합

### 3.4 입력 폼 UX 설계 (최종)
**목표**: 최소 입력 / 최대 재미, 모바일 기준 최적 UX, 로그인 없이 즉시 결과 제공

#### 입력 순서 (중요)
1. 동물 종류 선택
2. 반려동물 생년월일
3. 집사 생년월일
4. 이름 입력 (선택)
5. 결과 보기
(동물 → 반려동물 → 집사 순서가 가장 자연스러운 흐름)

#### 동물 종류 입력
- **타입**: 단일 선택 버튼 (고양이 🐱, 강아지 🐶, 기타 🐾)
- **규칙**: 하나만 선택 가능, 기본 선택 없음.
- **기타 선택 시**: 추가 입력 노출 "어떤 동물인가요?" (최대 10자)

#### 반려동물 생년월일 입력
- **타입**: 모바일 Date Picker
- **라벨**: 반려동물 생일
- **기능**: 날짜 선택 후 자동 나이 표시 (예: "현재 나이: 3세 2개월")
- **모름 옵션**: "모름 (평균 나이를 기준으로 분석)" 체크박스 제공
    - 체크 시: Date Picker 비활성화, 평균값(3세, 6월 1일) 자동 적용
    - 체크 해제 시: Date Picker 활성화

#### 집사 생년월일 입력
- **타입**: Date Picker
- **라벨**: 집사 생일
- **보조 문구**: "궁합 계산에 사용돼요"

#### 선택 입력 (몰입도 향상)
- 반려동물 이름 (선택 / 최대 10자)
- 집사 이름 (선택 / 최대 10자)
- 미입력 허용

#### CTA 버튼 UX
- **텍스트**: 궁합 결과 보기
- **활성화 조건**: 동물, 반려동물 생일, 집사 생일 모두 입력 완료
- **미완료 시**: "정보를 입력해주세요"

#### 로딩 UX (필수)
결과 계산 중 랜덤 문구 출력
- "궁합 계산 중…"
- "별자리 정렬 중…"
- "오행 분석 중…"
- "운명 데이터 해석 중…"

### 3.5 개인정보 안내 문구 (필수 삽입)
입력 폼 하단 고정:
> "입력하신 생년월일 정보는 결과 계산에만 사용되며 어떠한 개인정보도 저장되지 않습니다. 본 서비스는 로그인 없이 이용 가능합니다."

### 3.6 하단 안내 문구
> "본 콘텐츠는 엔터테인먼트 목적의 운세 서비스입니다."

## 4. 결과 페이지

결과 상단에 summary 문장을 대표 결과로 노출. 궁합 점수는 0~100 게이지 UI로 시각화.

### 4.0 결과 안내 문구 (조건부)
- **조건**: 반려동물 생년월일 '모름' 선택 시

# 개발 명세

## 1. 전체 아키텍처
- **구성**: Frontend (React) → Backend API (Node.js / Express) → Rule Engine (사주 계산 로직) → JSON 템플릿 데이터
- **특징**: AI API 사용 없음, DB 의존도 매우 낮음, 계산 중심 서비스

## 2. Frontend 기능 명세

### 2.1 페이지 구성
- `/pet-destiny`
- `/pet-destiny/result?seed={hash}`

### 2.2 입력 페이지 기능
- **State**: `ownerBirthDate`, `petBirthDate`, `petType` ("cat" | "dog" | "other")
- **검증**: 미래 날짜 불가, 오늘 기준 0세 이상만 허용
- **API 호출**:
    - `POST /api/pet-destiny`
    - Request: `{ "ownerBirth": "1989-10-25", "petBirth": "2018-08-25", "petType": "cat" }`
    - 성공 시: `/pet-destiny/result?seed={hash}` 로 이동. 결과 데이터는 React state 또는 localStorage에 저장.

### 2.3 결과 페이지 기능
- **출력 데이터**: `compatibility`, `personality`, `health`, `mind`, `lifetimeFlow`, `yearFortune`
- **공유 기능**: 카카오 공유 버튼 (결과 요약 텍스트 생성), URL 공유

## 3. Backend 기능 명세

### 3.1 API 목록
`POST /api/pet-destiny`
- 역할: 입력값 기반 운세 계산, 모든 결과 생성 후 반환

### 3.2 Request Schema

  "ownerBirth": "string (YYYY-MM-DD)",

  "petType": "cat" | "dog" | "other"

### 3.3 Response Schema

{
  "summary": "string",

  "personality": "string",
  "health": "string",

  "yearFortune": "string",
  "yearFortuneLabel": "string"

## 4. 서버 핵심 로직 구조

**서비스 레이어**:
- `services/zodiacService.js`
- `services/elementService.js`
- `services/compatibilityService.js`
- `services/healthService.js`
- `services/fortuneService.js`
- `services/seedService.js`

**처리 흐름**:
1. 연도 추출 -> 2. 천간 계산 -> 3. 오행 계산 -> 4. 궁합 계산 -> 5. 건강 운 계산 -> 6. 마음 경계 계산 -> 7. 평생 운 계산 -> 8. 올해 운 계산 -> 9. Seed 기반 문장 선택 -> 10. 결과 반환

## 5. 결과 캐싱 전략
- **캐시 Key**: `hash(ownerBirth + petBirth + petType)`
- **캐시 TTL**: 24시간
- **방식**: 서버 메모리 캐싱

## 6. DB 설계
- **DB 불필요 (MVP 기준)**: 모든 결과는 계산으로 생성, 저장 기능 없음.
- **데이터 소스**: 코드 내부 JSON 파일 사용 (`/data` 디렉토리 내 `personality.json`, `health.json`, `mind.json`, `compatibility.json`, `fortune.json`)
- **규모**: 200~500줄 텍스트

## 7. Seed 시스템
- 랜덤 사용 금지
- `seed = hash(ownerBirth + petBirth + petType)`
- `index = seed % template.length`

## 8. 보안 및 정책
- **저장 데이터 없음**: 로그인 없음, 개인정보 저장 없음.
- **서버 로그**: 생년월일 저장 금지.
- **허용 날짜**: 집사(1900~), 반려동물(2000~).
- **Rate Limit**: IP 기준 분당 10회.

## 17. 오행별 상세 성격 분석 (Detailed Personality Analysis)

오행 타입별 단순 키워드 나열이 아닌, **5~8문장의 자연스러운 서술형 성격 분석**을 제공한다.

### 17.1 생성 목표
- **엔터테인먼트 목적**: 사용자가 읽었을 때 "우리 아이랑 딱 맞네!" 라고 느낄 수 있는 공감가는 문장
- **구조**: 인트로 → 핵심 성격(긍정) → 사회성/행동 패턴 → 집사와의 관계 → 아웃트로
- **분량**: 5~8문장 (약 150~250자)

### 17.2 오행별 성격 가이드
| 오행 | 핵심 키워드 | 행동 특징 예시 |
|---|---|---|
| **목 (Tree)** | 호기심, 활동적, 장난 | "새로운 장난감을 보면 참지 못하며, 집안 곳곳을 탐험하는 것을 좋아합니다." |
| **화 (Fire)** | 애교, 감정표현, 관심욕구 | "집사의 관심을 독점하고 싶어하며, 기분이 좋으면 온몸으로 표현합니다." |
| **토 (Earth)** | 안정, 충성, 루틴 | "매일 같은 시간에 밥을 먹고 산책하는 규칙적인 생활을 선호합니다." |
| **금 (Metal)** | 깔끔, 예민, 독립, 관찰 | "자신만의 영역이 중요하며, 낯선 사람에게는 시간을 두고 천천히 다가갑니다." |
| **수 (Water)** | 지혜, 조용, 내향, 눈치 | "말하지 않아도 집사의 기분을 귀신같이 알아채고 조용히 곁을 지켜줍니다." |

### 17.3 생성 알고리즘 (Rule-based)
AI API를 사용하지 않고 결정론적(Deterministic)으로 문장을 조합한다.

1. **템플릿 풀 구성 (`personality_detail.json`)**:
   - 각 오행별로 `intro`, `body_positive`, `body_habit`, `relation_owner`, `outro` 카테고리별 문장 3~5개씩 보유
2. **문장 선택**:
   - `seed`를 사용하여 각 카테고리에서 문장을 하나씩 랜덤(Deterministic) 선택
   - `index = (seed >> shift) % length`
3. **치환**: `{name}` 플레이스홀더를 실제 반려동물 이름으로 치환
4. **조합**: 선택된 문장들을 자연스럽게 연결하여 하나의 단락으로 완성

**예시 출력**:
> "콩이는 안정적이고 충성스러운 성격을 가지고 있어요. 차분하고 신중하며, 쉽게 흥분하지 않아요. 집사에게 깊은 의리를 보여주며 언제나 든든하게 곁을 지켜줍니다. 콩이와 함께 있으면 마음이 편안해지고, 꾸준히 돌봐주고 싶은 마음이 생깁니다."
