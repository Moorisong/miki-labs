# 🎮 게임 로직 및 메카닉 (Game Mechanics)

## 1. 구현 원칙
* **목표**: 100% 물리 시뮬레이션보다는 "재미있는 조작감" 우선
* **React 연동**: 
  - React는 UI 오버레이(점수, 버튼 등) 관리
  - 실제 게임 루프와 상태는 `GameManager` (Non-React Class)에서 관리 권장

## 2. 상세 메카닉
### 집게 (Claw)
* **구조**: 3갈래 집게 (Three-pronged)
* **물리**:
  - `cannon-es`를 사용한 물리 연산
  - 집게의 악력(Strength)을 랜덤화하여 뽑기 확률 변동
  - 마찰 계수(Friction) : 인형마다 다르게 적용하여 잡기 난이도 조절

### 게임 플레이 흐름
1. **이동**: 사용자가 집게 위치 조작
2. **하강**: 버튼 클릭 시 집게 하강
3. **잡기**: 충돌 감지 및 물리엔진 기반 그립
4. **상승 및 이동**: 
   - 상승 중 집게 힘을 의도적으로 감소시키거나(놓침 연출) 유지
   - 입구까지 이동 시에도 흔들림 구현
5. **결과**: 입구 영역에 인형 도달 시 성공 판정

## 3. 카메라 (Camera)
* 집게의 위치를 추적하되 약간의 딜레이(Smoothing)를 주어 부드러운 느낌 제공
* 쿼터뷰 또는 탑뷰 시점 조정 가능하도록 구성
