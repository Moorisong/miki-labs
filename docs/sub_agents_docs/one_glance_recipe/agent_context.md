# 🤖 한눈 레시피 에이전트 가이드

## 1. 서비스 미션
사용자가 몇 번의 터치만으로 "전문가가 쓴 듯한 자연스러운 레시피"를 생성할 수 있도록 조력한다.

## 2. 구현 가이드라인
- **데이터 처리**: `ingredients`와 `steps`의 상태 변화를 실시간으로 감지하여 문장 생성 로직을 동기화한다.
- **문장 생성 로직 (Core)**: 
  - 조사(`을/를`, `와/과`) 처리 시 한글 종성(받침) 유무를 체크하는 유틸리티 함수를 반드시 사용할 것.
  - 조리 방법별 템플릿 배열을 관리하고, 중복을 피하기 위해 랜덤 선택 후 이전 값과 비교할 것.
- **UI 컴포넌트**: 
  - 재료 카드와 조리 단계 카드는 독립적인 컴포넌트로 분리한다.
  - 드래그 앤 드롭을 통한 순서 변경 지원을 고려한다.
- **LocalStorage**: 
  - `TTL` 로직을 위해 매 저장 시 현재 시간 + 72시간을 `expireAt`으로 설정한다.

## 3. 에러 핸들링 및 예외 상황
- 재료명 중복 입력 시 토스트 알림을 띄우고 추가를 막는다.
- 필수 옵션 누락 시 버튼을 비활성화하거나 기본값을 즉시 주입한다.
- 생성된 문장이 너무 짧거나 비어있을 경우 "자연스러움 체크" 단계에서 보정한다.

## 4. 기술 스택 참고
- 기존 하루상자의 디자인 시스템(CSS 변수 등)을 준수한다.
- 광고 연동 로직은 프로젝트 공통 유틸리티를 활용한다.
