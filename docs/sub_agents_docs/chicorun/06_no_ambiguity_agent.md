# 🚨 No Ambiguity Agent Reference

## 02_chicorun_planning.md (Section 8.5)
본 문서는 치코런 서비스의 문제 생성 시 복수 정답이나 모호한 선택지(Ambiguity)를 방지하기 위한 지침을 담고 있습니다.

---

## AI 작업 지침

### 목적
* 모든 생성 문제는 **단 하나의 명확한 정답**만 존재해야 함.
* 문법적으로 가능하지만 상황에 따라 정답이 될 수 있는 '부분 정답' 후보를 원천 차단함.

### 핵심 회피 패턴
1. **불가산 명사와 관사**: 
   - `history, water, homework, information` 등은 관사 유무에 따라 정답이 갈릴 수 있으므로, 관사 유무를 묻는 오답으로 사용하지 않음.
   - 예: `She studies history (O)` vs `She studies the history (X? 가능할 수도 있음)` -> 이 조합 자체를 생성 금지.
2. **시제 혼용**:
   - `She runs`와 `She is running`이 한 문제의 선택지에 동시에 나타나는 것을 금지 (시제 변환 문제 제외).
3. **맥락 의존 문장**:
   - `She opened the door` 처럼 `a` 인지 `the` 인지 맥락이 필요한 문장은 생성 지양.

### 검증 체크리스트
- [ ] 정답과 오답이 문법적으로 명확히 구분되는가?
- [ ] 원어민이 보기에 두 개 이상의 선택지가 정답이 될 가능성은 없는가?
- [ ] 불가산 명사에 불필요한 관사 오답이 붙어있지 않은가?
- [ ] 맥락 없이 읽었을 때 문장이 자연스러운가?

### 구현 로직 (참고)
* `UNCOUNTABLE_NOUNS` 리스트를 참조하여 관사 필터를 적용함.
* `hasConflict` 로직을 통해 시제 혼용 및 관사 모호성을 최종 단계에서 필터링함.
